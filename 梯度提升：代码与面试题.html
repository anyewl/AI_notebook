🚀 梯度提升算法 (GBDT) 深度解析与实战

💻 核心算法纯手工 PyTorch 实现

本代码使用 PyTorch 的张量操作直接复现了上述 HTML 中梯度提升的核心计算过程。没有调用任何 sklearn 树模型库，全程手工推演残差拟合与切分点查找，特别适合加深对张量计算和自动求导前置逻辑的理解。

import torch

# 1. 准备原始数据集 (复刻 HTML 中的演示数据)
# 特征 X: [学习时长 x1, 考勤率 x2]
X = torch.tensor([
    [10.0, 90.0], 
    [2.0, 50.0], 
    [5.0, 80.0], 
    [3.0, 60.0], 
    [8.0, 85.0]
], dtype=torch.float32)

# 真实分数 y
y = torch.tensor([85.0, 40.0, 65.0, 55.0, 75.0], dtype=torch.float32)

class RegressionStump:
    """CART 回归树桩 (仅进行一次特征切分的决策树)"""
    def __init__(self):
        self.split_feat = None
        self.split_val = None
        self.left_val = None
        self.right_val = None

    def fit(self, X, r):
        """寻找让加权 MSE 最小的切分点来拟合残差 r"""
        best_mse = float('inf')
        n_samples, n_features = X.shape

        # 遍历所有特征
        for feat_idx in range(n_features):
            unique_vals = torch.unique(X[:, feat_idx])
            # 遍历该特征下的所有可能切分值
            for val in unique_vals:
                # 使用 PyTorch 布尔掩码进行分割
                left_mask = X[:, feat_idx] <= val
                right_mask = X[:, feat_idx] > val

                # 忽略无效切分
                if left_mask.sum() == 0 or right_mask.sum() == 0:
                    continue

                # 切分残差
                left_r = r[left_mask]
                right_r = r[right_mask]

                # 叶子节点的输出最优权重 (当前桶内残差的平均值)
                mean_l = left_r.mean()
                mean_r = right_r.mean()

                # 计算左右子树的均方误差 (MSE)
                mse_l = torch.mean((left_r - mean_l)**2) if len(left_r) > 0 else 0
                mse_r = torch.mean((right_r - mean_r)**2) if len(right_r) > 0 else 0

                # 计算加权总 MSE
                total_mse = (len(left_r) / n_samples) * mse_l + (len(right_r) / n_samples) * mse_r

                # 记录最优切分点
                if total_mse < best_mse:
                    best_mse = total_mse
                    self.split_feat = feat_idx
                    self.split_val = val
                    self.left_val = mean_l
                    self.right_val = mean_r

    def predict(self, X):
        """根据切分点输出预测值"""
        preds = torch.zeros(X.shape[0], dtype=torch.float32)
        left_mask = X[:, self.split_feat] <= self.split_val
        right_mask = X[:, self.split_feat] > self.split_val
        preds[left_mask] = self.left_val
        preds[right_mask] = self.right_val
        return preds

class PyTorchGBDTRegressor:
    """梯度提升回归树 (手工 PyTorch 版)"""
    def __init__(self, n_estimators=2, learning_rate=0.8):
        self.n_estimators = n_estimators
        self.lr = learning_rate
        self.F0 = None
        self.trees = []

    def fit(self, X, y):
        # 步骤 1: 初始化基准预测值 F0 (总体均值)
        self.F0 = y.mean()
        F_m = torch.full_like(y, self.F0)

        # 步骤 2: 迭代训练多棵树
        for m in range(self.n_estimators):
            # 计算负梯度 (在使用 MSE 时，负梯度正好等于残差 r)
            r = y - F_m
            
            # 使用决策树桩拟合本轮残差
            tree = RegressionStump()
            tree.fit(X, r)
            self.trees.append(tree)

            # 计算新树的输出，并乘以学习率更新全局模型
            # 公式: F_m(x) = F_{m-1}(x) + lr * h_m(x)
            h_m = tree.predict(X)
            F_m = F_m + self.lr * h_m
            
            print(f"Tree {m+1} 已拟合 | 切分特征索引: {tree.split_feat} | 切分阈值: {tree.split_val.item():.1f}")
            print(f"当前轮次残差: {r.tolist()}")

    def predict(self, X):
        # 步骤 3: 累加所有树的预测结果
        F = torch.full((X.shape[0],), self.F0, dtype=torch.float32)
        for tree in self.trees:
            F = F + self.lr * tree.predict(X)
        return F

# ================= 测试与运行 =================
if __name__ == "__main__":
    print("--- 开始训练 GBDT 模型 ---")
    gbdt = PyTorchGBDTRegressor(n_estimators=2, learning_rate=0.8)
    gbdt.fit(X, y)

    print("\n--- 执行新样本预测 ---")
    test_X = torch.tensor([[6.0, 70.0]], dtype=torch.float32) # [学习时长=6, 考勤率=70]
    pred = gbdt.predict(test_X)
    print(f"对新样本特征 [6.0, 70.0] 的预测总分: {pred.item():.2f}")

📝 GBDT 核心概念突击面试题 (Top 5)

1. 为什么在 GBDT 中要拟合“负梯度”？它和“残差”到底是什么关系？
理论基础 重要性：⭐⭐⭐⭐⭐

关系：残差是负梯度在特定损失函数（均方误差 $MSE$）下的特例。当损失函数定义为 $L = \frac{1}{2}(y - F(x))^2$ 时，对其求负梯度的结果 $-\frac{\partial L}{\partial F(x)} = y - F(x)$，这就恰好等于真实值与预测值的差，即残差。

为什么拟合负梯度：在优化算法中，负梯度是函数值下降最快的方向。拟合负梯度意味着让模型总是朝着“让当前损失函数减小得最快”的方向去进化。这就使得 GBDT 具备了高度的通用性：只要更换不同的损失函数（如交叉熵、绝对误差等），只要能求导算梯度，这套提升框架就能工作，而不仅局限于算残差。

2. CART 回归树在寻找特征的最佳切分点时，使用的底层数学标准是什么？
算法细节 重要性：⭐⭐⭐⭐

标准是：最小化切分后的加权总均方误差 (MSE)。

算法会遍历每一个特征的所有可能取值作为切分点，将数据分为左右两组（子节点）。然后计算左子节点的内部 $MSE_L$ 和右子节点的内部 $MSE_R$。最终选定的切分点，必须使加权总误差 $MSE_{total} = \frac{N_L}{N} MSE_L + \frac{N_R}{N} MSE_R$ 达到全场最小。这保证了切分出来的叶子节点内部纯度最高，数据差异最小。

3. GBDT 建立一棵回归树后，叶子节点的最优输出权重 $b_{m,j}$ 是如何计算的？
数学推导 重要性：⭐⭐⭐⭐

最优权重是通过让该叶子节点内所有样本的损失最小化求解出来的。对于回归任务（MSE损失），我们要找一个常量 $b$ 最小化公式：$\sum_{i} \frac{1}{2}(r_i - b)^2$。

对其求导并令导数为 0 即可得出：$b_{m,j} = \frac{1}{N} \sum r_i$。换句话说，叶子节点的最优输出权重，就是落入该节点的所有样本当前残差的算术平均值。

4. 在 GBDT 的全局更新公式 $F_m(x) = F_{m-1}(x) + \eta \sum b_{m,j} I(x \in R_{m,j})$ 中，学习率 $\eta$ 的作用是什么？如果没有它会怎样？
模型调优 重要性：⭐⭐⭐⭐

作用：学习率 $\eta$（又称缩减系数 Shrinkage）相当于每一次模型迭代的“刹车”或步长控制，通常在 $0$ 到 $1$ 之间。它控制了每一棵新树对最终预测结果的影响力度。

没有它的后果：如果 $\eta = 1$，模型完全信任当前树拟合的残差，非常容易导致“步子迈得太大”从而快速出现过拟合。引入较小的学习率，意味着让模型小步慢跑，需要更多的树（更多的纠错接力）来逼近目标值，通常能带来更好的泛化能力。

5. 如果为了抵抗异常值的干扰，把 GBDT 的损失函数从均方误差 (MSE) 换成平均绝对误差 (MAE)，每一轮新树要拟合的目标会发生什么变化？
拓展思考 重要性：⭐⭐⭐⭐⭐

当损失函数变成 $MAE = |y - F(x)|$ 时，我们再去对其求关于 $F(x)$ 的导数，会发现负梯度不再等于残差的具体数值，而是等于残差的符号（Sign 函数）。

也就是说，如果预测低了（真实值 > 预测值），负梯度恒为 $1$；如果预测高了，负梯度恒为 $-1$。此时新树拟合的目标就不再是具体的误差大小了，而是只关心“方向”。这也正是 MAE 能够抗异常值的原因，因为即使某个异常样本误差高达 $10000$，在算法眼中它的负梯度也只是普通的 $1$ 而已，不会将树的生长方向带偏。
