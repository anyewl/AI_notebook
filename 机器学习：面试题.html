核心算法与工程面试题库

1L1与L2正则化的核心区别是什么？机器学习⭐⭐⭐⭐⭐

讲解：L1产生稀疏解（可用于特征选择），L2让权重趋于0但非0（平滑，防止过拟合）。优化角度上，L1是绝对值函数的梯度，在0点不可导且有恒定梯度；L2的梯度随权重减小而减小。

L = Ldata + λ∑|wi|      (L1)
L = Ldata + λ∑wi2       (L2)

扩展：从贝叶斯角度看，L1相当于为参数引入拉普拉斯先验，L2相当于引入高斯先验。

2偏差(Bias)与方差(Variance)的权衡关系？机器学习⭐⭐⭐⭐⭐

讲解：偏差衡量模型预测期望与真实值的偏离程度（欠拟合）；方差衡量训练集微小变化导致模型性能的变化（过拟合）。模型越复杂，偏差越小，方差越大。

Error = Bias2 + Variance + Noise

扩展：Bagging降低方差（如Random Forest），Boosting主要降低偏差（如GBDT）。

3逻辑回归的损失函数是什么？为什么要用它？机器学习⭐⭐⭐⭐

讲解：使用对数损失（Log Loss / Cross Entropy Loss）。如果不使用该损失而使用MSE，配合Sigmoid激活函数会导致损失函数非凸，容易陷入局部最优，且在预测值与真实值误差大时梯度消失。

L = -[y·log(p) + (1-y)·log(1-p)]

扩展：逻辑回归本质上是假设数据服从伯努利分布，通过极大似然估计(MLE)推导出来的损失函数。

4SVM的核心思想与核技巧(Kernel Trick)？机器学习⭐⭐⭐⭐

讲解：SVM旨在寻找一个超平面，使得正负样本之间的间隔(Margin)最大化。核技巧通过非线性映射将低维空间线性不可分的数据映射到高维空间使其线性可分。

K(x, z) = φ(x)Tφ(z)

扩展：常用的核函数包括线性核、多项式核以及高斯核(RBF)。高斯核可以映射到无限维。

5如何处理正负样本极度不平衡的情况？机器学习⭐⭐⭐⭐

讲解：数据层面可采用过采样（SMOTE）或欠采样；算法层面可调整损失函数权重（Class Weight）；评估层面绝不能使用Accuracy，应使用F1-Score、ROC-AUC或PR曲线。

扩展：在深度学习中，常用Focal Loss来解决类别极度不平衡及难易样本不平衡问题。

6K-Means算法的具体执行步骤？机器学习⭐⭐⭐

讲解：1. 随机初始化K个聚类中心。2. 计算每个样本到各中心的距离，将其分配给最近的簇。3. 重新计算每个簇的平均值作为新的中心。4. 重复2-3直至中心不再移动。

扩展：K-Means对初始值和异常值敏感。K的选取可通过手肘法(Elbow Method)或轮廓系数决定。

7K-Means++ 是如何改进初始化的？机器学习⭐⭐⭐

讲解：为了避免K-Means陷入局部最优，K-Means++优化了初始化：随机选取第一个中心后，后续选取的中心倾向于离现有中心越远越好（距离越大，被选为新中心的概率越大）。

P(x) = D(x)2 / ∑D(xi)2

扩展：虽然初始化时间变长，但收敛速度极大加快，整体性能优于随机初始化。

8PCA（主成分分析）的原理是什么？机器学习⭐⭐⭐⭐

讲解：PCA是一种线性降维方法，其目标是将数据投影到新的低维正交坐标系中，使得投影后的数据方差最大化（保留最多信息），或重构误差最小化。

扩展：求解过程本质上是对数据的协方差矩阵进行特征值分解，取前K个最大特征值对应的特征向量组成投影矩阵。

9KNN算法中K值大小有什么影响？机器学习⭐⭐⭐

讲解：K值较小：模型复杂，容易过拟合（高方差），对噪点敏感；K值较大：模型简单，容易欠拟合（高偏差），类别边界变模糊。

扩展：KNN没有显式的训练过程，属于惰性学习（Lazy Learning），通常使用KD树或球树加速查询。

10ROC曲线与AUC值的物理意义？机器学习⭐⭐⭐⭐⭐

讲解：ROC曲线以FPR为横轴，TPR为纵轴。AUC是ROC曲线下的面积。物理意义是：随机抽取一个正样本和一个负样本，模型给正样本打分高于负样本的概率。

扩展：ROC-AUC不受正负样本比例影响。但在极度不平衡且更关注少数正类时，PR曲线(Precision-Recall)更为敏锐。

11CART决策树如何进行分类与回归？树模型⭐⭐⭐⭐⭐

讲解：CART是一棵二叉树。分类时，使用基尼指数(Gini)选择特征，Gini越小表示纯度越高；回归时，使用均方误差(MSE)选择特征，叶子节点值为样本均值。

Gini(D) = 1 - ∑pk2

扩展：相比于多叉树，二叉树实现简单，且能通过多次分裂同一个连续特征来逼近复杂决策边界。

12C4.5算法是如何选择特征的？树模型⭐⭐⭐⭐

讲解：C4.5使用信息增益率(Information Gain Ratio)来选择特征。它克服了信息增益偏向于选择取值较多的特征（如ID字段）的缺点。

GainRatio(D, A) = Gain(D, A) / IV(A)

扩展：IV(A)为特征A的固有值(Intrinsic Value)，特征取值越多，IV值越大，从而起到惩罚作用。

13决策树的预剪枝与后剪枝有何区别？树模型⭐⭐⭐

讲解：预剪枝在树构建过程中，满足特定条件（如深度、最小样本数）就停止分裂，速度快但有欠拟合风险；后剪枝是先生成完全树，再自底向上评估合并叶子节点，精度高但耗时。

扩展：CART默认采用代价复杂度剪枝(Cost-Complexity Pruning)，通过引入正则化参数控制树的复杂度。

14Bagging与Boosting的区别是什么？树模型⭐⭐⭐⭐⭐

讲解：Bagging并行训练多个独立基分类器，通过投票或平均输出，旨在降方差（如随机森林）；Boosting串行训练，后续模型专注于纠正前面模型的错误，旨在降偏差。

扩展：因为Boosting已经通过加法模型降偏差，所以其基学习器通常是弱学习器（浅层决策树），以避免过拟合。

15随机森林(Random Forest)的"随机"体现在哪？树模型⭐⭐⭐⭐

讲解：体现在两点：1. 样本随机：使用自助采样法(Bootstrap)有放回抽样；2. 特征随机：每次节点分裂时，不遍历所有特征，而是随机选择子集特征寻找最优切分。

扩展：双重随机性极大地降低了模型间的相关性，从而进一步提升了Bagging降低总体方差的效果。

16GBDT的核心数学原理是什么？树模型⭐⭐⭐⭐⭐

讲解：GBDT的核心在于每棵树拟合的不是原始标签，而是当前模型在训练集上的负梯度（对于平方损失，负梯度就是残差）。通过不断拟合负梯度逼近真实分布。

rmi = - [∂L(yi, f(xi)) / ∂f(xi)]

扩展：GBDT只能使用回归树（CART回归），即使是分类任务，拟合的也是连续的对数几率值。

17XGBoost对GBDT做了哪些核心改进？树模型⭐⭐⭐⭐⭐

讲解：1. 损失函数：进行二阶泰勒展开，利用一阶(g)和二阶(h)导数；2. 正则化：在目标函数加入叶子数与叶子权重的L2正则项；3. 节点分裂：支持近似直方图算法与特征并行求最优切分点。

Obj = ∑[giwj + 0.5hiwj2] + γT + 0.5λ∑wj2

扩展：XGBoost还内置了处理缺失值的默认方向分配策略，无需对数据进行严格的前期插补。

18LightGBM相对XGBoost的优势是什么？树模型⭐⭐⭐⭐

讲解：1. 内存与速度：原生使用直方图算法(Histogram)，将连续特征离散化；2. 树分裂机制：使用带有深度限制的按叶子生长(Leaf-wise)策略，而非按层生长(Level-wise)；3. 互斥特征捆绑(EFB)与单边梯度采样(GOSS)。

扩展：Leaf-wise能获得更好的精度，但容易导致树过深引发过拟合，因此需要严格控制max_depth。

19树模型如何评估特征重要性(Feature Importance)？树模型⭐⭐⭐⭐

讲解：主要有三种指标：1. Weight(频次)：特征在所有树中作为分裂节点的总次数；2. Gain(增益)：该特征分裂带来的总信息增益（最常用）；3. Cover(覆盖度)：受该特征分裂影响的样本数比例。

扩展：基尼重要性虽然快，但可能偏向高基数特征。工程中常引入SHAP值(基于博弈论)进行更精确的特征归因。

20XGBoost是如何处理特征缺失值的？树模型⭐⭐⭐

讲解：在训练时，对于存在缺失值的特征，XGBoost会分别将缺失样本全部分配到左子节点和右子节点，计算两次增益，选择增益较大的一方作为该特征缺失值的默认分裂方向（Sparsity Aware Split）。

扩展：如果是预测时遇到训练集未见过的缺失特征，默认将其分配至右子节点。

21什么是梯度消失与梯度爆炸？深度学习⭐⭐⭐⭐⭐

讲解：在深层网络中，由于反向传播依据链式法则，连续相乘的梯度若小于1，传到浅层时会指数级衰减至0（消失）；若大于1，则会指数级放大（爆炸）。这导致浅层参数无法有效更新或NaN。

扩展：解决方法包括：合理的参数初始化（Xavier/Kaiming）、更换激活函数（ReLU）、残差连接（ResNet）、梯度裁剪（Clip）和BN层。

22Xavier初始化与Kaiming(He)初始化的适用场景及原理？深度学习⭐⭐⭐⭐⭐

讲解：两者目标都是保持输入输出的方差一致。Xavier适用于线性激活函数（如Sigmoid/Tanh的非饱和区），其方差缩放为 `2 / (n_in + n_out)`；Kaiming初始化考虑到ReLU会将负半轴清零（方差减半），因此将方差放大两倍，设为 `2 / n_in`。

Var(W) = 2 / nin      (Kaiming for ReLU)

扩展：使用Kaiming正态分布初始化时，标准差 σ = √(2/nin)。如果不对应激活函数使用，仍会引发梯度发散或消失。

23Dropout的工作原理及缩放机制？深度学习⭐⭐⭐⭐

讲解：训练时，随机让部分神经元以概率p停止工作，阻止神经元间的共适应。测试时，所有神经元生效，但输出需乘以 `(1-p)` 以保持期望一致。

扩展：PyTorch等框架使用Inverted Dropout，即在训练阶段将保留的激活值除以 `(1-p)`，这样测试阶段就无需任何缩放操作，提升推理速度。

24Batch Normalization(BN)的作用与计算公式？深度学习⭐⭐⭐⭐⭐

讲解：BN通过对小批量(Batch)的数据在特征维度上进行标准化，缓解内部协变量偏移(ICS)，允许使用更大的学习率。然后引入可学习参数进行还原。

y = γ · [(x - μ) / √(σ2 + ε)] + β

扩展：训练时计算当前Batch的均值方差，并更新全局移动平均；测试时必须使用全局移动平均（滑动平均）而非Batch统计量。

25为什么ReLU比Sigmoid更适合作为深层网络的激活函数？深度学习⭐⭐⭐⭐

讲解：1. 避免梯度消失：ReLU在正区间的导数恒为1；Sigmoid的导数最大只有0.25。2. 计算高效：ReLU只需判断是否大于0，无指数运算。3. 稀疏激活性：产生类似L1的稀疏表示。

ReLU(x) = max(0, x)

扩展：ReLU的缺点是"Dead ReLU"现象（负区梯度为0导致神经元死亡），可通过Leaky ReLU或参数初始化改善。

26Adam优化器相比于SGD有什么改进？深度学习⭐⭐⭐⭐⭐

讲解：Adam结合了动量(Momentum)和自适应学习率(RMSProp)。它计算梯度的一阶矩估计（动量，控制方向）和二阶矩估计（未中心化方差，自适应调整不同参数的学习率），并包含偏差修正。

θ = θ - α · [m_hat / (√v_hat + ε)]

扩展：SGD往往能在后期找到更平缓的泛化极小值，所以很多CV论文采用 SGD+Momentum 以及复杂的学习率衰减策略，而非Adam。

27交叉熵(Cross-Entropy)损失的推导与公式？深度学习⭐⭐⭐⭐

讲解：交叉熵衡量两个概率分布之间的差异。在分类任务中，真实分布y是One-hot向量，模型预测分布是p。最小化交叉熵等价于最大化正确类别的对数似然(MLE)。

CE = - ∑ yi · log(pi)

扩展：KL散度 D(y||p) = 交叉熵(y, p) - 信息熵(y)。由于真实标签的信息熵是常数，优化交叉熵就等于优化KL散度。

28Softmax函数的计算与数值稳定性问题？深度学习⭐⭐⭐⭐

讲解：Softmax将网络输出的Logits映射为和为1的概率分布。直接计算指数容易导致溢出(Overflow)。

Si = exi / ∑exj

扩展：工程实现中，通常会先求出Logits中的最大值 `C = max(x)`，然后每个元素减去C：`e^(x_i - C)`，从而防止数值溢出，且数学上等价。

29深度学习中局部最小值与鞍点(Saddle Point)的探讨？深度学习⭐⭐⭐

讲解：在高维非凸优化空间中，真正的局部最小值极其罕见。梯度为0的点绝大多数是鞍点（某个方向上是极小值，另一个方向上是极大值）。

扩展：Hessian矩阵在鞍点既有正特征值也有负特征值。优化器引入动量(Momentum)和随机噪声可以有效逃离鞍点区域。

30AdamW中的Weight Decay与L2正则化的区别？深度学习⭐⭐⭐⭐

讲解：在标准SGD中，L2正则化等价于Weight Decay（每次权重衰减一个常数比例）。但在Adam中，由于自适应学习率的分母会改变L2惩罚的缩放，直接用L2效果极差。AdamW将Weight Decay解耦，直接作用在参数更新步骤上。

扩展：这是为何在Transformer等大型模型中强制使用AdamW而非Adam的核心原因。

31PyTorch中的 `tensor.contiguous()` 与内存共享机制？PyTorch⭐⭐⭐⭐⭐

讲解：PyTorch的Tensor由元数据(Header)和一维的数据区(Storage)组成。`view()`、`transpose()` 只是改变元数据的步长(stride)，底层的Storage共享且未变化，导致内存不连续。`contiguous()` 会强制开辟新内存，重新排列数据使其在内存中连续存放。

扩展：`reshape()` 会自动判断：如果内存连续就等价于 `view()`，如果不连续就等价于 `contiguous().view()`。

32PyTorch的Autograd自动求导机制原理？PyTorch⭐⭐⭐⭐⭐

讲解：Autograd通过构建有向无环图(DAG)记录所有运算。Tensor的 `requires_grad=True` 且 `is_leaf=False` 时，运算会生成 `grad_fn` 节点。调用 `.backward()` 时，引擎利用链式法则从根节点向叶子节点传播梯度。

扩展：非叶子节点的梯度在反向传播结束后会被释放以节省内存。若想保留，需要调用 `.retain_grad()`。

33`model.eval()` 与 `with torch.no_grad()` 的区别？PyTorch⭐⭐⭐⭐

讲解：`model.eval()` 改变的是模型的模式：关闭Dropout，固定BatchNorm层的均值和方差，但它依然会构建计算图和消耗内存。`with torch.no_grad()` 改变的是引擎的行为：完全停止构建计算图，不保存前向激活值，极大降低显存消耗并加速推理。

扩展：在推理(Inference)阶段，这两者必须同时使用。

34DDP(DistributedDataParallel)为何优于DP(DataParallel)？PyTorch⭐⭐⭐⭐

讲解：DP是单进程多线程，存在Python GIL锁瓶颈，且每次前向传播主节点都要广播模型，反向传播要汇总梯度，造成负载不均。DDP是多进程，每个GPU有独立的模型副本和优化器，仅在反向传播时利用Ring-AllReduce同步梯度(Gradients)。

扩展：DDP显存利用率高、通信开销小，是PyTorch官方推荐的分布式训练方案。

35DataLoader中的 `num_workers` 和 `pin_memory` 作用？PyTorch⭐⭐⭐

讲解：`num_workers` 开启多个子进程异步预加载数据，避免GPU等待CPU读图。`pin_memory=True` 将数据锁定在物理内存（锁页内存）中，防止被操作系统换出到磁盘，从而使 CPU 到 GPU(VRAM) 的 DMA 拷贝速度翻倍。

扩展：`num_workers` 的值并非越大越好，过多会导致进程切换开销过大，一般设为CPU核心数或GPU数目的4倍。

36CNN中的感受野(Receptive Field)如何计算？计算机视觉⭐⭐⭐⭐

讲解：感受野指特征图上一点映射回原图的区域大小。计算公式是从后向前推导，当前层的感受野不仅依赖当前核大小，还依赖前一层的累积步长(Stride)。

RFi = RFi+1 + (Kernel_size - 1) * Stride_cumulative

扩展：膨胀卷积(Dilated Convolution)可以在不增加参数量的情况下指数级扩大感受野。

37U-Net在图像分割中表现出色的网络结构特点？计算机视觉⭐⭐⭐⭐⭐

讲解：U-Net采用对称的Encoder-Decoder架构。核心创新是跳跃连接(Skip Connection)，将下采样阶段的高分辨率空间信息直接拼接(Concat)到上采样阶段的深层语义特征上，精确定位像素边界。

扩展：全卷积网络(FCN)使用按元素加法(Add)，而U-Net使用通道拼接(Concat)，这使得它在医学图像等少样本精细分割任务中占据统治地位。

381x1卷积的核心作用是什么？计算机视觉⭐⭐⭐⭐

讲解：1. 跨通道信息融合：组合不同Feature Map的特征。2. 升维或降维：控制输出通道数，大幅减少计算量（如ResNet的Bottleneck模块）。3. 增加非线性：1x1卷积后接激活函数，增强表达能力。

扩展：在PointNet中，对点云进行的共享MLP处理，本质上等价于2D图像中的1x1卷积。

39推荐系统：矩阵分解(MF)是如何工作的？推荐系统⭐⭐⭐⭐

讲解：将稀疏的用户-物品评分矩阵 `R(M*N)` 分解为两个低维稠密矩阵的乘积：用户特征矩阵 `P(M*K)` 和物品特征矩阵 `Q(N*K)`。预测时直接求内积：`r_hat = p_u · q_i`。

扩展：传统的SVD无法处理严重缺失值且计算复杂度极高，工程中通常使用ALS（交替最小二乘法）或梯度下降法求解隐向量。

40推荐系统：双塔模型(Two-Tower)为何在召回层成为标配？推荐系统⭐⭐⭐⭐⭐

讲解：由于召回阶段需要从千万级库中极速筛选，复杂交叉网络延迟过高。双塔模型分别对User和Item进行特征编码，仅在最后进行点积(Dot Product)或余弦相似度计算。

扩展：Item塔的Embedding可以提前计算(离线)并存入FAISS等向量数据库，线上请求时只需实时计算User塔的向量进行KNN近似检索，兼顾了效果与极致的耗时。
