模型调优：L1 与 L2 正则化 (Regularization)

在训练神经网络时，模型为了“死记硬背”训练集，可能会让权重参数变得极其复杂和庞大，导致在测试集上表现极差（过拟合）。正则化就像是给模型的参数加上了“惩罚税”，强迫模型保持简单，从而提高泛化能力。

L1 Regularization (Lasso):
$$L = L_{data} + \lambda \sum_{i} |w_{i}|$$
L2 Regularization (Ridge / Weight Decay):
$$L = L_{data} + \frac{\lambda}{2} \sum_{i} w_{i}^{2}$$

一、 PyTorch 代码实现与核心解析

在 PyTorch 中，L2 正则化非常常用，已经被直接集成在优化器中。而 L1 正则化通常需要我们手动计算。以下是一段完整可运行的代码，对比演示了这两种实现方式：

import torch
import torch.nn as nn
import torch.optim as optim

# 1. 准备一个极其简单的模拟数据集
torch.manual_seed(42)
X = torch.randn(100, 5)  # 100个样本，5个特征
y = torch.randn(100, 1)  # 100个目标值

# 2. 定义一个简单的线性回归模型
class SimpleModel(nn.Module):
    def __init__(self):
        super(SimpleModel, self).__init__()
        self.linear = nn.Linear(5, 1)
        
    def forward(self, x):
        return self.linear(x)

model = SimpleModel()
criterion = nn.MSELoss() # 这是原始的 L_data (均方误差)

# ==========================================
# L2 正则化 (通过优化器的 weight_decay 实现)
# ==========================================
# weight_decay 参数就是公式中的 lambda。
# 在 PyTorch 中，给 SGD 或 Adam 设置 weight_decay，等同于在计算 L2 正则化。
optimizer_l2 = optim.SGD(model.parameters(), lr=0.01, weight_decay=1e-4)

# ==========================================
# L1 正则化 (需要手动在 Loss 中添加计算)
# ==========================================
optimizer_l1 = optim.SGD(model.parameters(), lr=0.01) # 注意：这里不设置 weight_decay
lambda_l1 = 1e-4 # 定义 L1 的惩罚力度

print("开始训练...")
for epoch in range(10):
    # --- 前向传播 ---
    predictions = model(X)
    loss_data = criterion(predictions, y) # 计算纯净的数据误差
    
    # --- 计算 L1 惩罚项 ---
    l1_penalty = torch.tensor(0.0, requires_grad=True)
    for name, param in model.named_parameters():
        if 'weight' in name: # 我们通常只对权重 w 进行正则化，不对偏置 b (bias) 做惩罚
            l1_penalty = l1_penalty + torch.norm(param, p=1) # 计算绝对值之和 (L1范数)
            
    # 最终的总 Loss = 数据误差 + L1惩罚项
    # 如果你想用 L2，其实就不需要加这一步，因为上面定义 optimizer_l2 时 weight_decay 已经在底层默默做了。
    total_loss = loss_data + lambda_l1 * l1_penalty
    
    # --- 反向传播与优化 ---
    optimizer_l1.zero_grad() # 清空之前的梯度
    total_loss.backward()    # 自动求导计算所有梯度
    optimizer_l1.step()      # 更新参数
    
    if epoch % 2 == 0:
        print(f"Epoch {epoch}: Total Loss = {total_loss.item():.4f}")

代码细节深度讲解：我们在干嘛？

  • `weight_decay` 是什么： 在 optimizer_l2 中，我们直接传入了 weight_decay=1e-4。在 PyTorch 底层计算梯度更新公式 $w = w - \alpha \cdot \nabla L$ 时，它会自动把它变成 $w = w - \alpha \cdot (\nabla L + \lambda w)$。这里的 $\lambda w$ 正是 L2 正则化项 $\frac{\lambda}{2} w^2$ 对 $w$ 求导后的结果。所以设置它就等于开启了 L2 正则化。
  • 为什么要遍历 model.named_parameters()： 在手动写 L1 正则化时，我们要把模型里所有的 $w$ 都抓出来求绝对值。这个方法可以让我们看到每个参数的名字（比如 linear.weight 和 linear.bias）。
  • 为什么过滤掉 bias： 代码中写了 if 'weight' in name:。这是一个重要的工程细节。偏置项 $b$ 只是让整体函数上下平移，并不会增加模型的“弯曲程度（曲率）”。惩罚 $w$ 能让模型变平滑从而抗过拟合，但惩罚 $b$ 毫无意义，反而可能导致模型无法拟合数据的真实均值。
  • torch.norm(param, p=1)： 这是一个计算“范数”的内置函数。p=1 表示 L1 范数，也就是把这个张量里所有元素的绝对值加起来 ($\sum |w_i|$)。它会自动接入 PyTorch 的动态计算图（即 requires_grad=True），所以在后续 backward() 时，它能完美地计算出梯度参与反向传播。

二、 核心面试题题库 (7道)

基础概念 重要性: ★★★★★
Q1: 什么是正则化？它为了解决机器学习中的什么问题？
A: 正则化是一种修改学习算法的技术，目的是减小泛化误差（模型在未知测试集上的误差），而不是减小训练误差。
它主要用来解决过拟合（Overfitting）问题。当模型过于复杂时，会记住训练数据中的噪声。正则化通过在损失函数中增加一个针对参数大小的“惩罚项”，限制模型的复杂度，迫使模型学习到更普遍、更平滑的规律。
对比分析 重要性: ★★★★★
Q2: 请简述 L1 正则化（Lasso）和 L2 正则化（Ridge）的核心区别是什么？
A:
1. 公式差异： L1 是惩罚权重参数的绝对值之和 ($\sum |w|$)；L2 是惩罚权重参数的平方和 ($\sum w^2$)。
2. 模型特征（最重要）： L1 会产生稀疏权重矩阵，它倾向于将不重要的特征权重直接变成 0，因此自带“特征选择”功能。L2 则倾向于让所有权重都变小，无限接近于 0 但不等于 0，让权重分布得更加平滑。
3. 抗干扰能力： L1 对异常值更鲁棒（稳健），因为绝对值的导数是常数；而 L2 对于绝对值大的权重惩罚极其严厉（二次方），对异常值更敏感。
数学直觉 重要性: ★★★★☆
Q3: 为什么 L1 正则化能导致参数稀疏（即很多权重变成0）？
A: 从几何空间来看，假设只有两个权重 $w_1, w_2$。
原始数据的损失函数等高线是一圈圈的椭圆。
L2 正则化的约束空间是一个圆 ($w_1^2 + w_2^2 \le C$)。椭圆与圆的切点（最优解）通常在某个坐标象限内，这意味着 $w_1, w_2$ 都有非零的值。
L1 正则化的约束空间是一个带有尖角的正方形/菱形 ($|w_1| + |w_2| \le C$)。由于菱形有突出的顶点（通常位于坐标轴上），损失函数的椭圆向外扩张时，极大概率会最先碰到菱形的顶点。而在顶点处，必然有一个维度的坐标是 0（例如碰到了 Y 轴的顶点，则 $w_1 = 0$）。这就导致了参数的稀疏性。
框架底层 重要性: ★★★★★
Q4: 在 PyTorch 的优化器中，`weight_decay` 参数本质上是在做什么？
A: `weight_decay`（权重衰减）本质上是在执行 L2 正则化。
在每次更新参数时，除了减去损失函数对参数的梯度外，还会额外减去参数自身乘以一个衰减系数（即 `weight_decay` 的值）。
数学表示：$w_{new} = w_{old} - lr \cdot \nabla L - lr \cdot \lambda \cdot w_{old}$。
这让参数在每次迭代中，都本能地向 0 的方向“衰减”一点点，防止参数变得过大。
工程实践 重要性: ★★★☆☆
Q5: 我们只对模型中的 Weight（权重 w）进行正则化，为什么通常不对 Bias（偏置 b）进行正则化？
A: 过拟合的本质是模型过于复杂，在函数图像上表现为剧烈的扭曲和波动。权重 $w$ 决定了特征之间的交互和函数的曲率/斜率，惩罚 $w$ 能有效压制函数的扭曲。
而偏置项 $b$ 仅仅控制函数在空间中的整体平移（上下/左右移动），不改变其形状或曲率。对 $b$ 进行惩罚不仅不能缓解过拟合，反而可能导致模型无法捕捉数据的整体平均偏移（即拟合不出截距），造成欠拟合。所以在实践中，通常将其过滤掉。
数据预处理 重要性: ★★★★☆
Q6: 在使用 L1/L2 正则化之前，是否必须对特征数据进行标准化/归一化？为什么？
A: 必须。 正则化是对所有的权重参数一视同仁地施加惩罚（比如用同一个 $\lambda$）。
如果数据没有缩放，量级小的特征（如 0.001）对应的权重往往非常大，量级大的特征（如 10000）对应的权重往往非常小。如果在这种情况下强行施加正则化惩罚，正则项会不成比例地剧烈打压量级小特征的权重，导致模型严重扭曲。经过标准化后，所有特征处于同一量纲，正则化的惩罚才会公平有效。
拓展进阶 重要性: ★★★☆☆
Q7: 什么是弹性网络（Elastic Net）？它的提出是为了解决什么问题？
A: 弹性网络是 L1 正则化和 L2 正则化的结合体。
公式包含了两者：$L = L_{data} + \lambda_{1} \sum |w| + \lambda_{2} \sum w^2$。
解决的问题： 单纯的 L1 正则化在处理存在高度相关性的特征时（共线性），通常会随机保留其中一个特征，而把其他相关特征权重全设为 0，这会导致信息丢失。而弹性网络结合了 L2 的平滑特性，不仅能实现特征稀疏化（L1的功劳），还能让高度相关的特征群体保持一致的权重（L2的功劳），综合表现更加稳健。
