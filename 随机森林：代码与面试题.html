第一部分：基于 PyTorch 张量的随机森林完整实现

在此实现中，我们完全摒弃了传统的循环列表操作，利用 torch.Tensor 处理数据计算。代码复刻了前置章节中的“考试预测”数据集 $X \in \mathbb{R}^{5 \times 3}, Y \in \{0, 1\}$，并手动实现 有放回抽样 (Bootstrap)、特征屏蔽 (Feature Sub-sampling) 以及 CART 基尼系数 (Gini Impurity) 的计算。

import torch

class PyTorchCARTNode:
    def __init__(self, is_leaf=False, result=None, split_feature=None, split_value=None):
        self.is_leaf = is_leaf
        self.result = result
        self.split_feature = split_feature
        self.split_value = split_value
        self.left = None
        self.right = None

class PyTorchRandomForest:
    def __init__(self, n_estimators=3, max_depth=2):
        self.n_estimators = n_estimators
        self.max_depth = max_depth
        self.trees = []
        self.features_used = []

    def _calc_gini(self, y):
        """计算一维张量 y 的基尼系数"""
        if len(y) == 0:
            return torch.tensor(0.0)
        # 获取类别分布概率
        counts = torch.bincount(y)
        p = counts.float() / len(y)
        return 1.0 - torch.sum(p ** 2)

    def _build_tree(self, X, y, available_features, depth):
        # 停止条件1：节点纯度 100%
        if len(torch.unique(y)) == 1:
            return PyTorchCARTNode(is_leaf=True, result=y[0].item())
        
        # 停止条件2：达到最大深度或无特征可用
        if depth >= self.max_depth or len(available_features) == 0:
            majority_class = torch.argmax(torch.bincount(y)).item()
            return PyTorchCARTNode(is_leaf=True, result=majority_class)

        best_gini = float('inf')
        best_split = None

        # 遍历可用特征和该特征下的所有取值
        for feat_idx in available_features:
            unique_vals = torch.unique(X[:, feat_idx])
            for val in unique_vals:
                left_mask = X[:, feat_idx] <= val
                right_mask = ~left_mask
                
                y_left = y[left_mask]
                y_right = y[right_mask]
                
                if len(y_left) == 0 or len(y_right) == 0:
                    continue
                    
                # 计算加权 Gini Index
                w_left = len(y_left) / len(y)
                w_right = len(y_right) / len(y)
                gini_index = w_left * self._calc_gini(y_left) + w_right * self._calc_gini(y_right)
                
                if gini_index < best_gini:
                    best_gini = gini_index
                    best_split = (feat_idx.item(), val.item(), left_mask, right_mask)

        if best_split is None:
            majority_class = torch.argmax(torch.bincount(y)).item()
            return PyTorchCARTNode(is_leaf=True, result=majority_class)

        feat_idx, val, left_mask, right_mask = best_split
        node = PyTorchCARTNode(split_feature=feat_idx, split_value=val)
        
        node.left = self._build_tree(X[left_mask], y[left_mask], available_features, depth + 1)
        node.right = self._build_tree(X[right_mask], y[right_mask], available_features, depth + 1)
        
        return node

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.trees = []
        self.features_used = []

        for _ in range(self.n_estimators):
            # 1. 样本扰动：有放回随机抽样 (Bootstrap)
            indices = torch.randint(0, n_samples, (n_samples,))
            X_sample = X[indices]
            y_sample = y[indices]

            # 2. 特征扰动：随机屏蔽特征 (这里随机保留 2 个特征)
            n_select = max(1, n_features - 1)
            feat_indices = torch.randperm(n_features)[:n_select]
            self.features_used.append(feat_indices)

            # 训练单棵决策树
            tree = self._build_tree(X_sample, y_sample, feat_indices, depth=0)
            self.trees.append(tree)

    def _predict_tree(self, node, x):
        if node.is_leaf:
            return node.result
        if x[node.split_feature] <= node.split_value:
            return self._predict_tree(node.left, x)
        else:
            return self._predict_tree(node.right, x)

    def predict(self, X):
        predictions = []
        for x in X:
            tree_preds = [self._predict_tree(tree, x) for tree in self.trees]
            # 少数服从多数 (Majority Voting)
            majority = torch.argmax(torch.bincount(torch.tensor(tree_preds))).item()
            predictions.append(majority)
        return torch.tensor(predictions)

第二部分：常见高频面试题库

1. 随机森林为什么需要进行“有放回的随机抽样（Bootstrap）”？如果无放回会怎样？
算法原理高频核心

核心目的：在确保个体决策树差异性的同时，不丧失全局分布特征。

  • 如果使用全集不抽样： 每棵树喂入的数据完全一致，最终长出的决策树将完全相同，集成算法退化为单模型，投票机制失效。
  • 如果使用无放回抽样： 数据集被切割成完全不相交的几个子集，导致每棵树都非常“偏科”，只能学到总体数据的局部特征，方差极大，极易欠拟合。
  • 有放回抽样： 大约有 $36.8\%$ 的样本在每次抽样中不会被抽到（OOB数据）。这既保证了每棵树训练样本的差异性（扰动），又保证了数据总体分布的基本面。
2. 请写出 CART 决策树中计算基尼指数（Gini Index）的数学公式，并说明它物理意义。
数学推导高频核心

对于含有 $K$ 个类别的数据集 $D$，其样本属于第 $k$ 类的概率为 $p_k$，基尼系数公式为：

$$ Gini(D) = 1 - \sum_{k=1}^{K} p_k^2 $$

当依据特征 $A$ 将数据集拆分为 $D_L$ 和 $D_R$ 两部分时，特征 $A$ 的加权基尼指数计算为：

$$ Gini\_Index(D, A) = \frac{N_L}{N} Gini(D_L) + \frac{N_R}{N} Gini(D_R) $$

物理意义：基尼系数反映了从数据集中随机抽取两个样本，其类别标签不一致的概率。Gini系数越小，数据集的纯度越高。CART树通过寻找使 $Gini\_Index$ 最小化的特征和阈值来进行节点分裂。

3. 随机森林和深度学习框架下的全连接神经网络（DNN）相比，优势和劣势分别是什么？
对比分析综合重要

随机森林优势：

  • 对特征缩放无感： 不需要对数据进行归一化或标准化。
  • 抗过拟合能力强： 通过 Bagging 机制天然抵抗过拟合，超参数极少，无需复杂的学习率和正则化调整。
  • 可解释性好： 能够轻易输出特征重要性（Feature Importance）。
  • 处理表格类（Tabular）离散数据： 在传统结构化维度数据集上表现通常优于神经网络。

神经网络（DNN）优势：

  • 表征学习能力： 能够自动从非结构化数据（图像、文本、语音）中提取连续的高维空间特征，而无需人工构建特征工程。
  • 端到端优化： 可以基于梯度下降和链式求导法则（Autograd），进行极高复杂度的连续可导函数映射。
4. 什么是 OOB（Out-of-Bag）数据？它在随机森林中有什么重要作用？
算法细节中频

在使用有放回抽样（Bootstrap）产生单个树的训练集时，每次抽样单个样本未被抽中的概率为 $(1 - \frac{1}{N})^N$。当 $N \to \infty$ 时，极限值为 $\frac{1}{e} \approx 0.368$。

这意味着总有约 $36.8\%$ 的数据没有参与特定一棵树的训练。这些被遗漏的数据被称为 包外数据（Out-of-Bag, OOB）。

重要作用：

  • 天然的验证集： 无需另外专门划分验证集，就可以直接用 OOB 数据对构建好的随机森林进行性能评估（计算 OOB 误差），这在数据量稀缺时极其宝贵。
  • 计算特征重要性： 通过对 OOB 数据中的某个特征加入随机噪声，观测 OOB 误差上升的幅度，即可评估该特征的重要性。
5. 为什么随机森林中不仅要对样本进行随机抽样，还要对特征进行随机屏蔽（随机特征选择）？
算法原理高频核心

这是为了进一步降低决策树之间的相关性，从而最大化群体投票的效用（降低方差）。

如果在每一轮不进行特征屏蔽（即所有特征对所有树开放）：假设数据集中有一个具有“统治力”的超强特征。那么即使训练样本被扰动了，几乎所有的决策树在根节点（或前几层）都会选择这个“超强特征”进行分裂。这将导致所有的树结构长得极其相似（高度相关）。

通过特征屏蔽（子集随机选择），我们强制一部分树在分裂时“蒙上眼睛”，看不见那个超强特征，逼迫它们去探索并利用其他次优特征。这样训练出来的树群才能做到真正的“和而不同”，有效避免由于特定强特征导致的局部最优，极大提升了泛化能力。
