🤖 KNN 算法 (K-Nearest Neighbors) 高级学习笔记

适合数学基础薄弱同学的直观交互式教程 • 从生活常识直击工程本质

一、 核心概念：物以类聚，人以群分

核心精髓： “近朱者赤，近墨者黑”。要预测一个未知样本的结果，只需要看看在特征世界里，离它最近的已知样本都是什么样子的。

💡 老师的大白话举例：
想预测你未来的薪资（未知样本），就找出你身边关系最密切的 5 个朋友（K=5）。如果他们全都是大厂大牛，那你大概率也差不到哪去！

KNN 算法三大步：

  1. 算距离： 求解当前【预测样本】和空间中【所有已知样本】的距离。
  2. 圈邻居： 严格按照距离从小到大排序，取出距离最近的 \( K \) 个样本。
  3. 做投票：
    • 分类问题： 少数服从多数！看看这 \( K \) 个邻居里哪种类别最多，预测样本就属于哪一类。
    • 回归问题： 齐心协力求平均！把这 \( K \) 个邻居的数值加起来求一个平均数，作为预测结果。

二、 距离的奥秘：如何定义“远近”？

数学公式只是把我们生活中的物理常识“符号化”了。别怕，我们一个一个来看：

1. 欧几里得距离

生活场景： 两点之间的直线距离。好比你有一架无人机，直接走两点连成的直线飞过去。

\( d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2} \)

* 勾股定理的延伸。如果有更多维度，就在根号里一直往后加平方项。

2. 曼哈顿距离

生活场景： 城市格子街道距离。你开着出租车，无法穿墙，只能老老实实沿街道横着走、竖着走。

\( d = |x_2 - x_1| + |y_2 - y_1| \)

* 也就是两点横向距离绝对值 + 纵向距离绝对值。

3. 切比雪夫距离

生活场景： 国际象棋中国王走的步数。国王可以横、竖、斜着走，走一步的物理距离虽然不同，但在棋盘上都算作1步。

\( d = \max(|x_2-x_1|, |y_2-y_1|) \)

* 距离取决于横向和纵向差值中，最大的那一个。

🌌 闵可夫斯基距离：距离的“终极变形金刚”

它不是一个新距离，而是一个通用的公式框架。通过调整参数 \( p \)，它可以自由切换成上面的任意一种距离：

\( d = \left( \sum_{i=1}^{n} |x_i - y_i|^p \right)^{1/p} \)
  • 当 \( p = 1 \) 时 ➡️ 自动脱下外套，变成 曼哈顿距离
  • 当 \( p = 2 \) 时 ➡️ 摇身一变，变成 欧几里得距离
  • 当 \( p \to \infty \) 时 ➡️ 极限变形，变成 切比雪夫距离

三、 问题分类：分类 vs 回归

许多初学者容易把这两个概念搞混。记住：核心区别在于你想要预测的目标结果的类型：

任务类型 目标值性质 大白话解释 经典案例 KNN在此时的决策
分类问题
(Classification)		 离散变量 结果是有限个互不相交的“标签/类别”。 预测明天是[晴天/雨天/阴天]
判断邮件是否为[垃圾邮件]		 少数服从多数：看看邻居里哪个类别最多。
回归问题
(Regression)		 连续变量 结果是一个具体的、无限可能的“数字”。 预测明天的[最高温度: 23.5℃]
预测某套房子的[价格: 450万]		 齐心协力求平均：把邻居的数值加起来求平均。

四、 K 值的选择：为什么推荐奇数？

K值的选择就像是照相机的焦距，太小或太大都会让画面模糊：

  • 为什么要选奇数？ 为了防止“平局”。如果是偶数（例如K=4），投票时可能会出现 2比2 的尴尬死局。奇数能保证总有一方胜出。
  • K 选得太小（比如 K=1）： 模型会过于敏感。如果最近的那个样本刚好是个异常噪声，模型就会直接被带偏。这在机器学习里叫过拟合（Overfitting）。
  • K 选得太大（比如 K=100）： 模型会变得极其迟钝。如果K大到和所有样本总量一样，那无论你点哪里，预测结果永远都是拥有样本数最多的那个类，失去了分类的意义。这叫欠拟合（Underfitting）。

五、 代码实战：在真实业务场景中应用 KNN

Scikit-Learn (sklearn) 是 Python 中极其流行的机器学习库，它把复杂的底层距离计算全封装好了。我们结合两个真实的工程场景来感受一下：

1. KNN 解决“分类问题”

🎯 业务场景：在线学习平台“考证通过率”预测

某在线教育平台想要预测新注册的用户能否通过最终的职业资格认证考试。平台分析发现，“每周自主在线学习时长”是决定能否通过的最关键核心特征。

数据严谨含义：
• 特征值 \(X\)（连续变量）：用户每周在平台的有效听课与做题时长（单位：小时）。
• 目标值 \(y\)（离散标签）：历史用户的真实考试结果。0 代表“未通过”，1 代表“顺利通过”。

历史训练数据集：

历史学生 ID 特征：每周学习时长 \(X\)（小时） 目标：考试结果 \(y\)（标签） 业务实际状态描述
学生 A0.50极少上线，考试不及格（未通过）
学生 B1.50偶尔看看，考试不及格（未通过）
学生 C2.81学习较为认真，顺利通过考试
学生 D3.81非常刻苦，高分通过考试

预测目标：新来了一位小明同学，他每周在平台学习 4.2 小时，预测小明最终能否通过考试？

from sklearn.neighbors import KNeighborsClassifier

def knn_classifier_demo():
    # 1. 实例化 KNN 分类器
    # 设置 n_neighbors=3，表示让算法寻找距离小明最近的 3 个历史学生样本
    estimator = KNeighborsClassifier(n_neighbors=3)
    
    # 2. 准备特征数据 X 
    # 注意：sklearn 要求特征数据必须是二维矩阵（即行和列），所以每个时长都用方括号包裹
    X = [
        [0.5],  # 学生 A
        [1.5],  # 学生 B
        [2.8],  # 学生 C
        [3.8]   # 学生 D
    ] 
    
    # 3. 准备目标标签 y (一维数组，与 X 的行一一对应)
    y = [0, 0, 1, 1] 
    
    # 4. 训练模型（让模型把历史数据读入内存，并在特征空间中排布好）
    estimator.fit(X, y)
    
    # 5. 业务预测：预测新学生（小明，每周学习 4.2 小时）的考试结果
    # 同样，输入的待预测特征也必须是二维矩阵形式 [[4.2]]
    target_student = [[4.2]]
    prediction = estimator.predict(target_student)
    
    print("--- 📊 分类业务预测结果 ---")
    print("小明的预测标签结果为 ->", prediction) 
    
    # 算法决策逻辑解读：离 4.2 最近的三个历史点是 3.8(1), 2.8(1), 1.5(0)。
    # 投票结果为 1票未通过，2票通过。少数服从多数，最终输出 [1]
    if prediction[0] == 1:
        print("💡 业务结论：该用户大概率能够【通过】考试。")
    else:
        print("💡 业务结论：该用户存在挂科风险，系统将自动触发提醒。")

knn_classifier_demo()

2. KNN 解决“回归问题”

🏢 业务场景：二手房交易平台“精准房价”估算

房产中介平台需要上线一个自动估价系统。一套二手房的价格不仅受到卧室数量的影响，还和它所在的楼层、房屋的面积密切相关。我们需要根据这三个特征预测一个连续的金额数值。

数据严谨含义：
• 特征值 \(X\)（多维特征空间）：包含3个特征维度，即 [卧室数量(个), 房屋所在楼层(层), 房屋面积(十平方米)]。
• 目标值 \(y\)（连续变量）：该二手房在市场的真实成交价格（单位：百万元）。

历史挂牌数据集：

历史房源 ID 特征1：卧室数（个） 特征2：所在楼层（层） 特征3：面积（十㎡） 完整特征向量 \(X\) 目标值 \(y\)：房价（百万元）
房源 1123[1, 2, 3]1.50 (150万)
房源 2256[2, 5, 6]2.80 (280万)
房源 33129[3, 12, 9]4.50 (450万)
房源 441812[4, 18, 12]6.20 (620万)

预测目标：新录入了一套房源特征为：3个卧室、位于11层、面积为100平米（10个十平米），对应特征向量为 [3, 11, 10]。请为这套新房源估算一个合理的挂牌售价。

from sklearn.neighbors import KNeighborsRegressor

def knn_regression_demo():
    # 1. 实例化 KNN 回归器
    # 设置 n_neighbors=2，代表寻找与新房源特征最相似的 2 套历史房源
    estimator = KNeighborsRegressor(n_neighbors=2)
    
    # 2. 准备多维特征矩阵 X（每行代表一套房子的三个属性）
    X = [
        [1, 2, 3],    # 房源 1
        [2, 5, 6],    # 房源 2
        [3, 12, 9],   # 房源 3
        [4, 18, 12]   # 房源 4
    ]
    
    # 3. 准备目标连续变量 y（每套房子对应的真实成交价）
    y = [1.5, 2.8, 4.5, 6.2] 
    
    # 4. 训练回归模型
    estimator.fit(X, y) 
    
    # 5. 业务预测：估算新房源 [3室, 11层, 100㎡] 的市场合理售价
    new_house = [[3, 11, 10]]
    predicted_price = estimator.predict(new_house)
    
    print("--- 🏢 回归业务预测结果 ---")
    print(f"新房源的估算价格为 -> {predicted_price[0]:.2f} 百万元")
    
    # 算法决策逻辑解读：计算发现新房源与 房源3[3,12,9] 和 房源4[4,18,12] 的空间距离最近。
    # 于是回归算法直接把这两个最相似房源的价格求平均值：(4.5 + 6.2) / 2 = 5.35
    print(f"💡 业务结论：该新房源推荐挂牌价为 {predicted_price[0] * 100:.0f} 万元。")

knn_regression_demo()
