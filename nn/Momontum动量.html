核心直觉： SGD（随机梯度下降）在遇到峡谷时，会因为梯度左右震荡而“左右横跳”，浪费大量时间。Momentum（动量）则引入了物理惯性，它会把过去几次的梯度方向累加起来，从而抵消震荡，加速向最优解前进。

🚙 通俗类比：越野车与碎石路 (Mini-batch 震荡)

为什么普通的 Mini-batch SGD 会震荡导致训练变慢？想象你开着一辆没有避震器的越野车走在坑洼的碎石路上。

  • 普通 SGD（无动量）： 就像一个“健忘”的司机，每一次遇到路面的小坑洼（Mini-batch 算出的局部噪声梯度），车轮就会猛打方向盘。结果就是车子在路两边来回画“Z”字型，虽然整体在往前走，但大部分力气都浪费在“左右横跳”上了。
  • Momentum（有动量）： 相当于给车子装上了重型飞轮。即使这一步车轮碾到了碎石想往右偏，但因为过去很长一段路积累了巨大的“向前惯性”，这块小碎石根本无法改变飞轮的大方向。前进方向变得平滑，车速自然就快了！

1. 动态演示：为什么 Momentum 更快？

🔴 红色 (普通 SGD)： 每次只看当前的一小批数据，遇到波动就疯狂抖动。

🟢 绿色 (Momentum)： 拥有惯性，自动平均并抵消了上下的震荡波动，路径平滑且高效。

2. 数学本质：指数到底藏在哪？

很多理论教材中的 EWMA 公式会带有一个 \((1-\beta)\)，但 PyTorch 的真实源码实现却去掉了它。看下面的对比，它们的数学本质是一致的：

理论课上的：指数加权平均

特点： 让均值的量级与当前梯度保持一致。

$$ S_t = \beta S_{t-1} + (1 - \beta) G_t $$
  • \(S_{t-1}\): 旧的指数加权均值
  • \(G_t\): 当前梯度
  • \(1 - \beta\): 抑制新数据的权重

PyTorch 源码中的：动量算法

特点： 当前梯度系数为 1！直接累加总惯性。

$$ v_t = \mu \cdot v_{t-1} + g_t $$ $$ w_{new} = w_{old} - \eta \cdot v_t $$
  • \(v_{t-1}\): 动量缓冲区 (Velocity)
  • \(g_t\): 当前梯度（系数永远是 1）
  • \(\eta\): 学习率（靠它来对冲放大的动量）
  • \(\mu\): 动量系数 (Momentum，如 0.9)

注：PyTorch 之所以省略 \((1-\beta)\)，是为了减少一次乘法计算。虽然动量 \(v_t\) 被放大了，但最终会通过乘以学习率 \(\eta\) 来控制步长，工程上更加高效。

3. 深度拆解：PyTorch 代码与 8.0 是怎么算出来的？

我们来看一段极简的 PyTorch 代码，并手动还原底层优化的第一步数学推导：

import torch
import torch.optim as optim

# 假设模型参数 w
w = torch.tensor([10.0], requires_grad=True)

# 定义优化器，momentum=0.9，学习率=0.1
optimizer = optim.SGD([w], lr=0.1, momentum=0.9)

# 模拟一步更新
optimizer.zero_grad()
loss = w ** 2  # 模拟 Loss = w^2
loss.backward()
optimizer.step()

print(f"更新后的参数: {w.item()}")
# 输出结果将是: 8.0

🧮 揭秘：控制台输出的 8.0 从何而来？

在执行 optimizer.step() 时，底层发生了以下三次计算：

  1. 求当前梯度 (\(g_t\))：
    已知 \(Loss = w^2\)，对 \(w\) 求导得 \(\frac{\partial Loss}{\partial w} = 2w\)。
    代入当前参数 \(w = 10.0\)，得到当前真实梯度：\(g_t = 2 \times 10.0 = 20.0\)。

  2. 更新动量缓冲区 (\(v_t\))：
    代入 PyTorch 公式 \(v_t = 0.9 \cdot v_{t-1} + g_t\)。
    因为这是第一步，过去没有惯性（\(v_{t-1} = 0\)），直接塞入新梯度：\(v_t = 0.9 \times 0 + 20.0 = 20.0\)。

  3. 正式更新参数 (\(w_{new}\))：
    代入公式 \(w_{new} = w_{old} - lr \cdot v_t\)。
    \(w_{new} = 10.0 - (0.1 \times 20.0)\)
    \(w_{new} = 10.0 - 2.0 =\) 8.0

4. 面试高频问题进阶

问：如果我在 PyTorch 里把 `dampening=1`（阻尼设为1）会发生什么？
答：PyTorch 包含阻尼的完整公式是：\(v_t = \mu \cdot v_{t-1} + (1 - \text{dampening}) \cdot g_t\)。
如果设为 1，公式就变成了 \(v_t = \mu \cdot v_{t-1}\)。这意味着当前的梯度被完全抛弃了！ 模型将像一个瞎子一样，完全忽略眼前的路况，只依靠之前的惯性盲目滑行，最终动量衰减为 0 导致训练彻底停滞。所以除了特殊实验，千万别这么设。
问：Momentum 遇到大平原（平缓区）有什么用？
答：在平缓区，普通 SGD 因为当前梯度极小，会陷入“龟速爬行”。而 Momentum 因为保留了之前冲下山坡的巨大惯性 \(v_{t-1}\)，哪怕当前梯度 \(g_t\) 接近 0，它依然能借助飞轮的惯性高速冲过这片平原。
