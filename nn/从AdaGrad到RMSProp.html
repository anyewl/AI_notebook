🚀 深度学习优化算法：从 AdaGrad 到 RMSProp

同学你好！学习 AI 就像是教一个蒙着眼的人下山（寻找损失函数的最低点）。梯度下降就是他用脚感受坡度（计算梯度）。如果感觉坡度很陡，他就迈大步；感觉平缓，就迈小步。这个“步伐的基础大小”，我们叫它学习率（$\eta$）。

🤔 小白的痛点：如果只用固定的基础步伐，可能遇到坑坑洼洼的地方会走不好。我们需要一种方法，让 AI 能够根据历史走过的路况，自动调整每一步的大小。这就是优化器要干的活！

一、AdaGrad：记仇的下山者

AdaGrad 的核心思想是：如果某个方向以前走得很快（累计坡度大），现在就让它慢一点；如果以前走得很慢，现在就给它一个大步伐。

🏃 形象比喻： 想象一个非常“记仇”的下山者，他有一个小本本，每次走一步，就把这一步的坡度平方后加到本子上。 在决定下一步跨多大时，他会用基础步伐除以本子上所有数字的总和的平方根。 这意味着：随着他走的路越来越远，本子上的数字只增不减。最后数字大到离谱，他的步伐就会变得无限小，导致他还没走到山底就累得停在半路了。
📐 核心公式解析：
1. 累计历史梯度的平方：$$S_t = S_{t-1} + g_t^2$$ （$g_t$ 是当前的坡度/梯度，$S_t$ 就是那个不断变大的小本本账单）

2. 计算当前的真实学习率并更新：$$\text{真实学习率} = \frac{\eta}{\sqrt{S_t + \sigma}}$$ （$\eta$ 是初始设定的基础学习率，比如 1.0；$\sigma$ 是一个极小的数比如 $1e^{-10}$，防止分母为0。可以看到，因为 $S_t$ 一直变大，导致真实学习率越来越小。）

二、RMSProp：懂得放下的下山者

因为 AdaGrad 后期会停滞不前，AI 大佬们提出了 RMSProp。既然历史包袱太重，那我们就引入遗忘机制！

🐟 形象比喻（金鱼记忆法）： 这个下山者也记账，但他只看重最近的路况，很久以前的路况会被他慢慢按比例“遗忘”。他每次记账时，会把旧账本乘上一个小于 1 的衰减系数（比如 0.9），再加上当前新坡度的 0.1。这样账本的数字就不会无限膨胀，步伐也就不会归零了！
📐 核心公式解析：
1. 使用指数加权移动平均（听起来高大上，其实就是衰减的旧账+打折的新账）： $$S_t = \beta \cdot S_{t-1} + (1 - \beta) \cdot g_t^2$$ （$\beta$ 就是遗忘系数，通常设为 0.9。代表旧账保留 90%，新账只占 10%。这让 $S_t$ 保持在一个平稳的范围。）

2. 更新公式（和 AdaGrad 长得一模一样，只是 $S_t$ 不再无限增大了）： $$\text{真实学习率} = \frac{\eta}{\sqrt{S_t + \sigma}}$$

四、💡 常见面试题解析（核心考点）

Q1：简述 AdaGrad 的核心机制，它在实际训练中最大的缺陷是什么？

解答：AdaGrad 的核心是实现了自适应学习率。它通过不断累积所有历史梯度的平方和（$S_t$）来约束每个参数的基础学习率。这意味着，频繁更新的参数分母较大、学习率小；不常更新的参数分母小、学习率大。这使得它非常适合处理稀疏特征。
最大缺陷：由于累加公式 $S_t = S_{t-1} + g_t^2$，随着迭代的进行，历史梯度的平方和会单调递增，导致分母越来越大，最终真实学习率（$LR$）会快速衰减至趋于 0。这会使得模型在还未收敛到全局最优解前，就提前停止了学习。

Q2：RMSProp 算法是如何改进 AdaGrad 以解决其缺陷的？

解答：RMSProp 创造性地引入了指数加权移动平均 (EMA) 机制。在更新历史累积变量时，它不再简单粗暴地累加所有历史梯度平方，而是使用衰减系数（$\beta$）来保留近期的真实地形情况，并平滑掉久远的历史记录。公式变为 $S_t = \beta \cdot S_{t-1} + (1 - \beta) \cdot g_t^2$。这样既保持了独立调整各维度学习率的自适应特性，又防止了 $S_t$ 的无限膨胀，彻底解决了学习率过早消失的问题。

Q3：在这些优化算法（包括 Adam）的公式中，分母中加上的 $\sigma$（或 $\epsilon$）的作用是什么？

解答：该参数是一个极小值的正常数（通常取值如 $1e^{-8}$ 级别）。它唯一的数学和工程作用是维持数值稳定性，防止分母计算为 0（避免 Zero Division Error）。如果该值设置过大，会反向主导分母，削弱甚至抵消自适应学习率的作用；如果过小，在某些计算框架或遭遇极端小梯度时，仍可能存在浮点数计算精度溢出的风险。

Q4：对比标准的固定学习率算法（如标准 SGD），自适应学习率算法优势在哪里？

解答：标准 SGD 在所有参数维度上使用全局一致学习率。而在复杂的深度神经网络的非凸优化问题中，不同维度的“地形曲率”差异常常十分巨大——在平缓方向上需要较大步长加速，而在陡峭方向上需要小步长以防震荡。自适应算法能够基于每个参数维度的各自梯度历史，自动地、独立地为每个参数分配其专属的动态学习率，从而大幅度提升了高维优化环境中的优化效率和收敛稳定性。

五、💻 PyTorch 代码实战

理论结合实践！下面是使用 PyTorch 实现 AdaGrad 和 RMSProp 优化器的基础代码示例。你可以直接将这些代码复制到你的 Python 环境中运行体验。

import torch
import torch.nn as nn
import torch.optim as optim

# 1. 定义一个简单的目标函数模型
# 我们直接优化一个张量，寻找函数 f(x, y) = x^2 + 2y^2 的最小值 (显然在 x=0, y=0 处取得)
class SimpleModel(nn.Module):
    def __init__(self):
        super().__init__()
        # 初始化参数为 [5.0, 5.0]
        self.params = nn.Parameter(torch.tensor([5.0, 5.0]))

    def forward(self):
        # 目标函数：f(x, y) = x^2 + 2y^2
        return self.params[0]**2 + 2 * (self.params[1]**2)

# ==========================================
# 演示 1：AdaGrad 优化器实战
# ==========================================
print("=== 开始 AdaGrad 优化 ===")
model_adagrad = SimpleModel()
# lr=1.0 对应上方公式里的基础学习率 eta，eps 对应防止分母为0的 sigma
optimizer_adagrad = optim.Adagrad(model_adagrad.parameters(), lr=1.0, eps=1e-8)

for step in range(5):
    optimizer_adagrad.zero_grad() # 清空历史梯度
    loss = model_adagrad()        # 计算当前 loss
    loss.backward()               # 反向传播计算梯度
    optimizer_adagrad.step()      # 根据 AdaGrad 规则更新参数
    
    print(f"Step {step+1}: 参数={model_adagrad.params.data.numpy()}, Loss={loss.item():.4f}")

# ==========================================
# 演示 2：RMSProp 优化器实战
# ==========================================
print("\n=== 开始 RMSProp 优化 ===")
model_rmsprop = SimpleModel()
# alpha=0.9 对应上方公式里的遗忘系数 beta
optimizer_rmsprop = optim.RMSprop(model_rmsprop.parameters(), lr=1.0, alpha=0.9, eps=1e-8)

for step in range(5):
    optimizer_rmsprop.zero_grad()
    loss = model_rmsprop()
    loss.backward()
    optimizer_rmsprop.step()
    
    print(f"Step {step+1}: 参数={model_rmsprop.params.data.numpy()}, Loss={loss.item():.4f}")
