二分类交叉熵 (BCE Loss) 教学演示

想象你在训练一个AI判断图片是不是“猫”。真实结果(y)只有两种：是(1)或不是(0)。预测概率(\(\hat{y}\))是AI给出的0到1之间的信心指数。

1. 核心公式与本质

\( L = -[ y \cdot \ln(\hat{y}) + (1 - y) \cdot \ln(1 - \hat{y}) ] \)
公式解码：
  • \( y \): 真实标签。如果图片是猫，\( y=1 \)。
  • \( \hat{y} \): 预测概率。AI觉得是猫的信心值，比如 0.9。
  • \( -\ln(\hat{y}) \): 核心惩罚机制。这部分是我们要重点看的地方。

本质：如果 \( y=1 \)，公式简化为 \( L = -\ln(\hat{y}) \)。当 AI 预测概率 \( \hat{y} \) 越接近 0（猜错了），损失 \( L \) 就会飞速飙升，从而迫使 AI “吸取教训”。

2. 数据案例对比 (匹配 PyTorch 代码数据)

样本 真实标签 (y) AI预测 (\(\hat{y}\)) 计算过程 损失 (Loss)
样本 1 0 0.3 \( -\ln(1-0.3) \) 0.3567
样本 2 1 0.8 \( -\ln(0.8) \) 0.2231
样本 3 0 0.1 \( -\ln(1-0.1) \) 0.1054
平均 (Total) - - (0.3567 + 0.2231 + 0.1054) / 3 0.2284

4. 开发者必看：PyTorch 实现细节

在 PyTorch 中进行二分类任务时，注意数据类型的设置至关重要：

# 1. 创建样本真实值
y_true = torch.tensor(data=[0, 1, 0], dtype=torch.float32)         # 【严重警告】：二分类索引编码必须使用 float32
print('Index Encoding must use dtype->', y_true.dtype)
# y_true = torch.tensor(data=[0, 1, 0], dtype=torch.int32)         # 使用 int32 会导致计算损失时【报错】
# y_true = torch.tensor(data=[0, 1, 0], dtype=torch.int64)         # 使用 int64 会导致计算损失时【报错】

# 2. 创建模型预测概率值
y_pred = torch.tensor(data=[0.3, 0.8, 0.1])

# 3. 创建二分类交叉熵损失函数
criterion = nn.BCELoss()                                            # 默认：reduction='mean'

# 4. 计算损失值
loss = criterion(y_pred, y_true)
print('loss ->', loss)  # 0.2284

5. 面试高频考题

Q1: 为什么分类任务中偏好使用交叉熵 (BCE) 而不是均方误差 (MSE)？

答：在使用 Sigmoid 激活函数时，MSE 的梯度在误差较大时会趋近于 0（梯度饱和），导致模型难以收敛。而交叉熵求导后，梯度与误差是线性关系，能够有效避免梯度消失问题。

Q2: 为什么在 PyTorch 中使用 BCELoss 时标签必须是 float？

答：因为 BCE 公式涉及对数运算和概率乘法，这在数学上定义于实数域。如果传入整数（int），会导致框架在做除法或梯度计算时因为类型不匹配而报错。PyTorch 要求输入和标签的 dtype 必须一致。
