💻 核心算法：CART 特征二分切分的 PyTorch 实现

虽然决策树传统上使用 Scikit-Learn 等基于 NumPy/Cython 的库，但通过 PyTorch 的张量（Tensor）操作，我们可以非常清晰地实现 CART 树的核心逻辑：Gini 不纯度计算与穷举二分。这种张量化计算也是深入理解深度学习框架底层逻辑的极佳方式。

以下代码完整复现了之前 HTML 中关于“天气”特征（晴天、多云、下雨）的 Gini 指数加权计算过程。

import torch

def calculate_gini(y):
    \"\"\"计算 Gini 不纯度，使用 Tensor 操作确保高效\"\"\"
    if len(y) == 0:
        return torch.tensor(0.0)
    
    # 获取各类别出现的频次
    _, counts = torch.unique(y, return_counts=True)
    probabilities = counts.float() / len(y)
    
    # 公式: Gini = 1 - sum(p_k^2)
    gini = 1.0 - torch.sum(probabilities ** 2)
    return gini

def evaluate_split(X, Y, feature_idx, split_value):
    \"\"\"评估按特定特征的特定值进行二叉切分的效果\"\"\"
    # 左子树: 特征等于 split_value (例如: == 多云)
    left_mask = (X[:, feature_idx] == split_value)
    # 右子树: 特征不等于 split_value (例如: == 晴天 或 下雨)
    right_mask = ~left_mask

    y_left = Y[left_mask]
    y_right = Y[right_mask]

    gini_left = calculate_gini(y_left)
    gini_right = calculate_gini(y_right)

    # 计算加权 Gini 指数
    weight_left = len(y_left) / len(Y)
    weight_right = len(y_right) / len(Y)

    gini_split = weight_left * gini_left + weight_right * gini_right
    return gini_split

# ==========================================
# 数据集构造 (对应露营案例)
# ==========================================
# X: [天气, 温度]
# 天气映射: 0=晴天, 1=多云, 2=下雨
# 温度映射: 0=低, 1=高
X = torch.tensor([
    [0, 1],
    [0, 0],
    [1, 1],
    [1, 0],
    [2, 0],
    [2, 1]
], dtype=torch.float32)

# Y标签: 0=不去, 1=去
Y = torch.tensor([1, 0, 1, 1, 0, 0], dtype=torch.float32)

print("--- PyTorch CART 天气特征切分测试 ---")
weather_idx = 0
weather_conditions = {0: "晴天", 1: "多云", 2: "下雨"}

for val, name in weather_conditions.items():
    gini = evaluate_split(X, Y, weather_idx, val)
    print(f"按 {name} vs 其他 切分, 加权 Gini = {gini.item():.4f}")

# 预期输出:
# 按 晴天 vs 其他 切分, 加权 Gini = 0.5000
# 按 多云 vs 其他 切分, 加权 Gini = 0.2500
# 按 下雨 vs 其他 切分, 加权 Gini = 0.2500

🎯 机器学习工程师高频面试题（CART相关）

基础理论 ⭐⭐⭐⭐⭐ 1. CART 树和传统的 ID3 / C4.5 决策树在结构上最本质的区别是什么？

解答：最本质的区别在于树的形态与分支规则。CART（Classification and Regression Trees）树是一棵严格的二叉树，它在任何节点上只能有“是”和“否”两个分支；而 ID3 树在遇到具有多个类别的分类特征时（如天气有3种状态），会直接产生多叉树（分出3个分支）。通过强制使用二叉结构，CART 简化了节点逻辑，也一定程度上避免了数据切分过细的问题。

核心机制 ⭐⭐⭐⭐⭐ 2. CART 算法既可以做分类也可以做回归，它在寻找切分点时，分类树和回归树使用的损失函数分别是什么？

解答：

  • 分类树： 使用基尼系数（Gini Impurity）来选择最优切分特征和切分点。寻找使得切分后两个子集合基尼系数加权和最小的方案。
  • 回归树： 使用均方误差（MSE, Mean Squared Error）准则，通过最小二乘法进行切分。寻找使得切分后左右两个子集各自的平方误差之和最小的方案，子节点的预测值通常为该节点内样本真实值的平均值。
数学公式 ⭐⭐⭐⭐ 3. 请写出计算基尼系数 (Gini Impurity) 的数学公式，并说明当数据集完全纯净时其值为多少？

解答：

假设数据集 $D$ 中包含 $K$ 个互斥的类别，第 $k$ 个类别在数据集中出现的概率（比例）为 $p_k$，则基尼系数的公式为：

$$Gini(D) = 1 - \sum_{k=1}^{K} p_k^2$$

当数据集完全纯净（即所有样本都属于同一个类别）时，某一个 $p_k = 1$，其余均为 $0$。此时 $Gini(D) = 1 - 1^2 = 0$。Gini 指数越小，代表数据越纯净。

算法细节 ⭐⭐⭐⭐ 4. 当 CART 树处理一个拥有 N 种离散属性的特征（且 N > 2）时，它是如何划分子集的？

解答：因为 CART 必须维持二叉树结构，它不能直接分出 N 个叉。它的做法是将 N 种属性的所有可能分为两个互斥的子集。具体来说，它会采用“穷举法”，把某一个属性单独拿出来作为左子集，剩下的所有属性合并作为右子集（或者更复杂的组合方式）。接着计算所有这些可能组合的加权基尼系数，最终选择能让基尼系数降到最低的那一种二分组合进行切分。

过拟合应对 ⭐⭐⭐ 5. 为什么 CART 树在生成后通常需要进行剪枝（Pruning）操作？它的剪枝策略叫什么？

解答：决策树在完全生长的情况下，非常容易记住训练集中的噪声和特异点，导致极其严重的过拟合（Overfitting）现象，模型变得异常庞大且在测试集上表现糟糕。因此需要剪掉一些收益不大的分支。

CART 采用的策略称为代价复杂度剪枝（Cost-Complexity Pruning）。该策略在评估是否剪枝时，不仅考虑模型对训练数据的拟合误差，还引入了一个正则化项（树的叶子节点个数）作为惩罚。它会自底向上地计算各个子树的表面误差率增益，如果剪掉某根树枝带来的误差增加能够被模型复杂度的降低所弥补，算法就会执行剪枝操作。
