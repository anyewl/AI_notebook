机器学习数据预处理：中位数填补与 IQR 异常值截断

架构说明： 在工业级深度学习或混合建模管道中，虽然 PyTorch 主要作为神经网络框架使用，但通过其强大的张量（Tensor）计算引擎进行数据预处理，可以无缝对接后续的 GPU 训练，避免 CPU 与 GPU 之间频繁的数据拷贝。本指南展示如何完全基于 PyTorch 实现特征工程中经典的中位数填补（Median Imputation）与 IQR 四分位距法（Interquartile Range）边界截断。

一、 PyTorch 完整可运行代码

preprocessing_pipeline.py
import torch

def create_dummy_data():
    '''
    创建一个包含缺失值(NaN)和极端异常值的PyTorch张量数据集。
    特征0: 包含NaN值，以及一个极大的正异常值。
    特征1: 包含NaN值，以及一个极小的负异常值。
    '''
    # 模拟 10 行, 2 列 的特征矩阵
    data = [
        [1.2,  10.5],
        [2.1,  12.3],
        [float('nan'), 11.1],  # 特征0 缺失
        [1.8,  14.2],
        [2.5,  float('nan')],  # 特征1 缺失
        [1.9,  13.0],
        [150.0, 12.5],         # 特征0 出现极大异常值 (150.0)
        [2.3,  -85.0],         # 特征1 出现极小异常值 (-85.0)
        [2.0,  11.8],
        [1.7,  12.1]
    ]
    return torch.tensor(data, dtype=torch.float32)

def main():
    # 设定随机种子及打印配置
    torch.manual_seed(42)
    torch.set_printoptions(precision=4, sci_mode=False)
    
    # 1. 初始化原始数据
    X_raw = create_dummy_data()
    print("=== 1. 原始张量数据 (包含 NaN 与 异常值) ===")
    print(X_raw)
    print("-" * 50)

    # ==========================================
    # 核心步骤一：基于 PyTorch 的中位数填补缺失值
    # ==========================================
    X_imputed = X_raw.clone()
    
    # 使用 torch.nanmedian 计算每一列忽略 NaN 后的中位数
    # torch.nanmedian 返回一个 namedtuple (values, indices)
    medians = torch.nanmedian(X_imputed, dim=0).values
    print(f"=== 2. 计算得到的各列中位数 ===")
    print(f"特征0 中位数: {medians[0].item():.4f}, 特征1 中位数: {medians[1].item():.4f}")
    
    # 检测缺失值掩码 (Mask)
    nan_mask = torch.isnan(X_imputed)
    
    # 按列填补缺失值
    for col_idx in range(X_imputed.shape[1]):
        col_nan = nan_mask[:, col_idx]
        X_imputed[col_nan, col_idx] = medians[col_idx]
        
    print("\n=== 3. 中位数填补缺失值后的张量 ===")
    print(X_imputed)
    print("-" * 50)

    # ==========================================
    # 核心步骤二：基于 IQR 四分位距法的边界截断
    # ==========================================
    X_cleaned = X_imputed.clone()
    num_features = X_cleaned.shape[1]
    
    # 分别计算 25% (Q1) 和 75% (Q3) 的分位数
    # 注意：torch.quantile 要求输入中不能包含 NaN 值，因此必须在填补后执行
    q1 = torch.quantile(X_cleaned, 0.25, dim=0)
    q3 = torch.quantile(X_cleaned, 0.75, dim=0)
    
    # 计算 IQR 
    iqr = q3 - q1
    
    # 计算上下边界截断点 (1.5倍 IQR)
    lower_bounds = q1 - 1.5 * iqr
    upper_bounds = q3 + 1.5 * iqr
    
    print("=== 4. IQR 异常值边界计算结果 ===")
    for i in range(num_features):
        print(f"特征 {i}: Q1={q1[i].item():.4f}, Q3={q3[i].item():.4f}, IQR={iqr[i].item():.4f}")
        print(f"      下界(Lower Bound) = {lower_bounds[i].item():.4f}")
        print(f"      上界(Upper Bound) = {upper_bounds[i].item():.4f}")
    
    # 使用 torch.clamp 动态截断边界外的异常值
    # 遍历每一列，将小于 lower_bound 的值置为 lower_bound，大于 upper_bound 的值置为 upper_bound
    for col_idx in range(num_features):
        X_cleaned[:, col_idx] = torch.clamp(
            X_cleaned[:, col_idx], 
            min=lower_bounds[col_idx].item(), 
            max=upper_bounds[col_idx].item()
        )
        
    print("\n=== 5. 经过 IQR 边界截断后的最终清洗数据 ===")
    print(X_cleaned)
    print("=" * 50)

if __name__ == "__main__":
    main()

💡 Python 代码深度解析与技术细节

  • 为什么使用 torch.nanmedian？ 在原生 PyTorch 中，如果对含有 NaN 的张量直接调用 torch.median，其返回结果将会是 NaN。自 PyTorch 1.10+ 起引入了 torch.nanmedian，它会自动过滤掉指定维度的 NaN 元素，从而准确计算出正常数据样本的中位数。
  • torch.quantile 的输入约束： 细心的读者会发现，代码中是先填补缺失值，再计算 IQR 边界。这是因为 torch.quantile 内部算子无法原生支持 NaN 值的忽略。若张量中带有缺失值，会触发运行时错误。因此在顺序上，必须将填补作为截断的前置任务。
  • 非原地操作与 clone() 的健壮性： 在预处理流水线中，直接修改原始张量（In-place 操作）容易引发计算图回溯或数据混乱风险。代码在每步均采用 X_raw.clone() 进行深度拷贝，确保各预处理阶段数据独立。
  • 高效边界截断 torch.clamp： 传统的 Python 循环或 Pandas 的 .apply() 效率较低。我们利用 PyTorch 原生高度优化的 C++ 算子 torch.clamp(input, min, max)，能在一行内完成整列数据的双向边界过滤与值替换。这种方式可无缝迁移至 GPU（只需调用 .cuda()），在处理千万级海量数据时，速度较 Pandas 提升数倍至数十倍。

二、 7道核心高频面试题精解

1. 在缺失值填补中，什么场景下应该首选中位数（Median）而不是均值（Mean）？
类型：核心原理与数理统计 重要性：⭐⭐⭐⭐⭐ (极高)
标准回答：
均值（Mean）对极端值和异常值极其敏感。当数据分布呈现明显的偏态分布（Skewed Distribution）或者数据中存在未处理的极端异常值时，均值会被拉向偏态的长尾一侧，无法真实反映数据的集中趋势。

相比之下，中位数（Median）是一个稳健统计量（Robust Statistic），其崩溃点（Breakdown Point）高达 50%，这意味着只要异常值样本占比不超过一半，中位数就能保持稳定。因此，在房价、人均收入等高度偏态或包含长尾噪声的机器学习任务中，应当优先选择中位数填补。
2. 详细阐述 基于 IQR（四分位距）法处理异常值的数学原理，为什么乘数通常设为 1.5？
类型：算法原理 重要性：⭐⭐⭐⭐ (高)
标准回答：
IQR 法通过计算数据的第三四分位数（$Q_3$）与第一四分位数（$Q_1$）的差值来确定数据的核心波动范围，即：
$$IQR = Q_3 - Q_1$$
其定义的有效数据区间为：$\left[ Q_1 - 1.5 \times IQR, \ Q_3 + 1.5 \times IQR \right]$。

为什么是 1.5？ 这一设定源于经典统计学对正态分布（Gaussian Distribution）的假设。在标准正态分布中，$Q_1 \approx -0.6745\sigma$，$Q_3 \approx 0.6745\sigma$，由此推导出的 $IQR \approx 1.349\sigma$。
当乘数取 $1.5$ 时，边界对应的位置约为 $\pm 2.698\sigma$。根据正态分布概率，此时落在该区间之外的概率仅为 0.7% 左右（即大约 $3\sigma$ 的覆盖范围）。这在统计学中被认为是一个合理识别低概率异常事件的阈值。若需要更宽松的阈值，通常会选择 $3.0 \times IQR$（称为极端异常值边界）。
3. 在工业级端到端模型部署中，如何正确应用中位数和 IQR 边界？需要注意什么工程隐患？
类型：工程实践 重要性：⭐⭐⭐⭐⭐ (极高)
标准回答：
核心要点在于严防数据泄露（Data Leakage）。
  1. 训练阶段： 必须且只能在训练集（Train Set）上计算出各特征的 median、Q1、Q3 以及 IQR 边界值，并将这些统计标量持久化保存起来（如保存为 JSON 或模型元配置文件）。
  2. 测试/推理阶段： 当线上新样本（Test Set 或实时 API 请求）流入时，绝对不能重新计算线上样本的中位数和 IQR，而是直接加载训练集留存的统计量，对新样本执行同样的填补和 clamp 截断。
工程隐患： 如果在生产环境对输入的小批量（Batch）或单条测试样本动态计算中位数，不仅会导致推理结果随着 Batch 的大小而发生剧烈波动，造成线上线下表现不一致，更违反了测试集不可见的原则。
4. 边界截断（Truncation/Clipping）与直接删除包含异常值的样本，各有什么优缺点？
类型：模型调优与策略对比 重要性：⭐⭐⭐⭐ (高)
标准回答：
  • 边界截断（如 torch.clamp）：
    • 优点： 最大程度保留了样本量，避免因某一列特征异常而损失掉该样本在其他维度上的关键特征信息，维持了数据集的完整性。
    • 缺点： 会在上下边界处人为造成“数据截断结”（即大量数据堆积在边界点上），改变了原始特征的连续分布形态，可能会对某些依赖平滑分布的参数模型带来微弱偏置。
  • 直接删除样本：
    • 优点： 保证了清洗后存留数据的纯净度，消除了噪声。
    • 缺点： 若数据属于多维稀疏型，单个样本多列异常可能导致级联删除，从而大幅削减训练样本规模。此外，若异常本身蕴含着业务信号（如欺诈交易检测），直接删除会导致模型完全丧失对该类长尾场景的捕获能力。
5. 缺失值中位数填补后，对下游线性模型（如逻辑回归）和树模型（如 XGBoost）的训练分别会带来什么潜在影响？
类型：模型综合与机制分析 重要性：⭐⭐⭐⭐⭐ (极高)
标准回答：
  • 对线性模型： 线性模型依靠特征全局连续变化来拟合权重。单一的中位数填补会人为降低该特征的方差，并在中位数点产生高浓度聚集，这会干扰线性权重的准确拟合。通常需要配套引入一个二值化的“缺失值指示器特征（Missing Indicator）”（即新建一列 0/1 特征标识其原本是否缺失），以告知模型此处的特定含义。
  • 对树模型： 现代高级树模型（如 XGBoost、LightGBM）内部自带缺失值分裂机制（稀疏感知算法），它们会自动将缺失值分配到增益最大的左/右子树中。如果我们强行采用中位数进行前置填补，反而破坏了树模型对缺失信号的自然捕获，阻碍了其根据“缺失”这一行为进行独立节点分裂的能力，有时甚至会带来模型性能的轻微下降。
6. 既然异常值处理很重要，为什么在深度神经网络（DNN）中很少看到显式执行 IQR 边界截断的代码？
类型：深度学习理论 重要性：⭐⭐⭐ (中)
标准回答：
深度学习对数值的容忍机制与传统浅层机器学习不同：
  • 神经网络内部拥有批归一化（Batch Normalization, BN）和层归一化（Layer Normalization, LN）等机制，它们通过动态重缩放特征，能够在前向传播过程中自动平抑部分极端输入引发的激活值膨胀。
  • 神经网络广泛使用非线性激活函数（如 Sigmoid、Tanh，甚至是具有硬截断或软饱和特性的 Clip/ReLU6），其底层逻辑天然包含了对大数值输入的非线性压缩。
因此，神经网络通常只需进行基础的标准化（Z-Score）或最小最大缩放（MinMax Scaler），而无需显式依赖 IQR 这类基于硬断裂边界的经典统计规则。
7. 假设在训练过程中，希望将缺失值填补和异常值截断作为端到端网络的一部分参与反向传播，这在 PyTorch 中可行吗？为什么？
类型：深度学习与工程架构 重要性：⭐⭐⭐⭐ (高)
标准回答：
不可行，且没有实际算法意义。
从数学求导的角度来看：
  1. 中位数的选择是通过对离散元素进行排序并提取中间索引实现的（不可导算子，其梯度在绝大多数地方为 0）；
  2. 分位数的计算（IQR 基础）属于非连续的分段线性选择，其无法提供能够优化前向输入分布的有效连续梯度；
  3. 缺失值检测 torch.isnan 返回的是一个布尔型掩码（Boolean Mask），离散的布尔算子天然无法参与反向传播。
因此，中位数填补和 IQR 异常值处理在架构设计上被牢固归类为前置静态数据清洗（Data Preprocessing）。如果要在网络内部动态处理缺失值并传递梯度，应当采用支持全微分的可学习组件，如 GAIN (Generative Adversarial Imputation Nets) 或在图神经网络（GNN）中利用连续的空间聚合机制。
