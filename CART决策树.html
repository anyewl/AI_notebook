CART 决策树进阶：如何把“三选一”逼成“二选一”？

当特征拥有 3 种以上属性时，看看 CART 是如何穷举组合，维持其完美的二叉树形态的。

1. CART 树的“人设”与核心特点

CART 的全称已经暴露了它的野心：它既能做分类（预测类别，如猫/狗/猪），也能做回归（预测连续数值，如房价、气温）。它有两个极其严格且鲜明的特点：

  • 严格的二叉树： 无论面对什么特征，CART 树在每一个节点上只会分出两条岔路（左边是“是”，右边是“否”）。它绝对不会像 ID3 那样一口气分出三四个杈。这种“非黑即白”的二分法让它的结构非常清晰，计算效率极高。
  • 先野蛮生长，再理智修剪： CART 树在训练时，倾向于把自己长得非常庞大、深邃（尽可能拟合所有训练数据），然后再通过一种叫“代价复杂度剪枝”的技术，把那些没用的细枝末节砍掉，以防止过拟合。

2. 它是如何做决策的？（核心数学原理）

既然 CART 是个“双面手”，它在处理分类和回归时，心里拨算的“算盘”是完全不同的：

场景 A：作为“分类树”时 —— 追求“纯度” (Gini 系数)

当你要预测的是离散类别（比如：用户是否会购买），CART 寻找最佳切分点的依据是基尼系数（Gini Impurity）。基尼系数衡量的是“杂质”或者“混乱度”。假设你闭着眼睛从一个袋子里随便抓两个球，这两个球颜色不一样的概率，就是基尼系数。如果袋子里全是红球，你抓出两个颜色不一样的概率是 0（绝对纯净，Gini = 0）。

假设数据集 $D$ 中有 $K$ 个类别，第 $k$ 个类别占的比例为 $p_k$，基尼系数的数学公式为：

$$Gini(D) = 1 - \sum_{k=1}^{K} p_k^2$$

它的切分逻辑： 遍历所有特征的所有可能切分点，算一算切完之后，左右两边子集的基尼系数之和能不能降到最低。谁能让分类后的数据最“纯”，就按谁来切。

场景 B：作为“回归树”时 —— 追求“稳定” (均方误差 MSE)

当你要预测的是连续数值（比如：用户的终身价值是 1500 元还是 15000 元），基尼系数就没法用了。此时，CART 会切换武器，使用平方误差最小化准则（最小二乘法）。

假设一个节点内有 $N$ 个样本，每个样本的真实值是 $y_i$，这个节点给出的预测值通常是这 $N$ 个样本真实值的平均数（记为 $\bar{y}$）。均方误差的计算公式为：

$$MSE = \frac{1}{N} \sum_{i=1}^{N} (y_i - \bar{y})^2$$

它的切分逻辑： 寻找一个切分点，使得切出来的左半边数据各自都很接近左半边的平均值，右半边数据各自都很接近右半边的平均值。换句话说，它要让切分后两个子集的方差之和最小。

全新的数据集：引入 3 种天气

我们对露营数据进行了升级，现在“天气”特征拥有 3 种可能：晴天、多云、下雨。

编号天气 (3种可能)温度去露营？(标签 Y)
1晴天高去
2晴天低不去
3多云高去
4多云低去
5下雨低不去
6下雨高不去

全局统计：总计 6 条数据。3 天去，3 天不去。

① 计算全局基尼不纯度 $Gini(Y)$

因为去和不去的概率是一样的（各占一半），这是系统最混乱的状态，基尼指数应该达到最大值。

$P(去) = \frac{3}{6} = 0.5$
$P(不去) = \frac{3}{6} = 0.5$
$$ Gini(Y) = 1 - (0.5^2 + 0.5^2) = 1 - 0.5 = 0.5 $$

结论：初始不纯度为 0.5（非常混乱）。

② 核心揭秘：CART 是如何切分【天气】特征的？

面对【晴天、多云、下雨】三个类别，CART 不会像 ID3 那样分三叉。它会把类别分为两个子集，进行“穷举试探”。对于 3 个类别，一共有 3 种二分法：

  • 方案一：问 “天气是多云吗？” $\rightarrow$ 切成 {多云} 和 {晴天, 下雨} 两个阵营。
  • 方案二：问 “天气是下雨吗？” $\rightarrow$ 切成 {下雨} 和 {晴天, 多云} 两个阵营。
  • 方案三：问 “天气是晴天吗？” $\rightarrow$ 切成 {晴天} 和 {多云, 下雨} 两个阵营。

接下来，CART 会老老实实地把这 3 种方案的基尼指数全都算一遍！

方案一：按 {多云} vs {晴天, 下雨} 切分

分支 A (是多云)： 包含编号 3, 4。
结果是 2 个“去”，0 个“不去”。
$Gini(多云) = 1 - (1^2 + 0^2) = 0$ (完美纯净！)
分支 B (不是多云)： 包含晴天(1,2)和下雨(5,6)。共 4 条数据。
结果是 1 个“去”(编号1)，3 个“不去”(编号2,5,6)。
$Gini(非多云) = 1 - \left( (\frac{1}{4})^2 + (\frac{3}{4})^2 \right) = 1 - \left( \frac{1}{16} + \frac{9}{16} \right) = \frac{6}{16} = 0.375$
方案一的加权基尼指数：
$$ Gini_{方案一} = \frac{2}{6} \times 0 + \frac{4}{6} \times 0.375 = 0 + 0.25 = 0.25 $$

方案二：按 {下雨} vs {晴天, 多云} 切分

分支 A (是下雨)： 包含编号 5, 6。全部是“不去”。
$Gini(下雨) = 0$
分支 B (不是下雨)： 包含晴天(1,2)和多云(3,4)。
结果是 3 个“去”(编号1,3,4)，1 个“不去”(编号2)。
$Gini(非下雨) = 1 - \left( (\frac{3}{4})^2 + (\frac{1}{4})^2 \right) = 0.375$
方案二的加权基尼指数：
$$ Gini_{方案二} = \frac{2}{6} \times 0 + \frac{4}{6} \times 0.375 = 0.25 $$

方案三：按 {晴天} vs {多云, 下雨} 切分

分支 A (是晴天)： 包含编号 1, 2。1 个“去”，1 个“不去”。
$Gini(晴天) = 1 - (0.5^2 + 0.5^2) = 0.5$ (最不纯净)
分支 B (不是晴天)： 包含多云(3,4)和下雨(5,6)。2 个“去”，2 个“不去”。
$Gini(非晴天) = 1 - (0.5^2 + 0.5^2) = 0.5$
方案三的加权基尼指数：
$$ Gini_{方案三} = \frac{2}{6} \times 0.5 + \frac{4}{6} \times 0.5 = 0.5 $$

结论：经过比对，方案一和方案二的基尼指数最小（都是 0.25），而方案三是 0.5。算法会随机在方案一或二中选一个作为根节点。假设 CART 选择了方案一（天气 == 多云？）。

③ 生成严格的二叉树图解

通过上面的计算，我们成功把拥有 3 种状态的“天气”变成了“是/否”的二分问题。左边直接纯净结案，右边由于混合了【晴天】和【下雨】，Gini > 0，会在下一层继续提问，直到彻底分清。

总结：不管一个特征有多少种取值，CART 永远秉持着“非黑即白”的原则，通过将特征划分为两个集合，算出所有组合情况的基尼指数，挑出最优解，从而生成一棵结构极其规整的二叉树。
