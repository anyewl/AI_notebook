为什么准确率会失效？

深入理解数据不平衡环境下的分类指标陷阱与分子稀释效应

核心机制总结

1. 基础概率陷阱 (Base Rate Fallacy)

当数据极端不平衡时（例如正负类比例 $1:99$），数据集本身自带一个高达 99% 的“先验分布”。

此时，模型即使退化成一个“零智能”的统计机器（不学习任何特征，拒绝做任何判断，直接盲猜所有输入都是负类），也能白嫖这 99% 的基础概率作为自己的准确率。高准确率反映的不再是模型的分辨能力，而是数据集的天然屏障。

2. 分子权重的“绝对稀释”

回顾准确率的标准数理公式：

$$Accuracy = \frac{TP + TN}{N}$$

该公式赋予了每一个样本绝对相等的权重。在欺诈检测或病理诊断中，我们真正关心的是那 1% 的正样本（即 $TP$ 能抓出多少）。

但由于 $TN$（真负例）的数量过于巨大，它在分子上形成了一种强烈的“稀释效应”。即使模型把正样本全部漏掉（$FN$ 极大，$TP = 0$），在庞大的 $TN$ 掩护下，整体得分依然坚挺。

二、 什么是 F1 的核心意义？

在机器学习中，精确率 ($P$) 和召回率 ($R$) 是一对天生的“冤家”。

  • 精确率 ($P$) 追求“宁缺毋滥”：只要我报警，就必须是真有贼。为了提高 $P$，模型会变得极其保守，只抓最有把握的。这会导致大量真实的贼被漏掉。核心思想：不考虑预测真假中的占比！
    • $$P = \frac{TP}{TP + FP}$$
  • 召回率 ($R$) 追求“宁错杀不放过”：只要有一丁点可疑，我就报警。为了提高 $R$，模型会变得极其敏感。这会导致满天都是假警报。核心思想：在所有真实样本中的占比！
    • $$R = \frac{TP}{TP + FN}$$

F1 的意义，就是做一个严厉的“裁判”。它要求模型必须在 $P$ 和 $R$ 之间取得平衡。如果你想通过走极端（比如让 $P=1.0$，但 $R$ 烂得一塌糊涂）来骗取高分，F1 裁判就会直接给你打不及格。

三、 什么叫“调和平均”？为什么要用它？

我们平时最熟悉的是算术平均 (Arithmetic Mean)，公式是 $\frac{P + R}{2}$。算术平均的特点是“和稀泥”，两个指标可以互相弥补。

调和平均 (Harmonic Mean) 的公式是：$\frac{2}{\frac{1}{P} + \frac{1}{R}}$ ，化简后就是 F1 的标准公式：

$$F1 = \frac{2PR}{P+R}$$

为什么必须用调和平均？我们来看一个最经典的物理例子：平均速度。

假设你从宿舍去实验室，距离是 60 公里：

  • 去程：你打车，速度极快，$v_1 = 60$ km/h。花费 1 小时。
  • 回程：你骑共享单车，速度极慢，$v_2 = 20$ km/h。花费 3 小时。

现在问：你这趟往返的平均速度是多少？

如果你用算术平均：$\frac{60 + 20}{2} = 40$ km/h。但这是错的！ 真正的平均速度 = 总路程 / 总时间 = $120 / 4 = 30$ km/h。

仔细观察这个真实结果 $30$：它离慢速的 $20$ 非常近，离快速的 $60$ 非常远。物理本质是：你在慢速上花费了整整 3 个小时，在快速上只爽了 1 个小时。慢速具有压倒性的“拖拽效应”。

在这个例子中，真正的平均速度 30，恰恰就是 60 和 20 的调和平均数！

$$HM = \frac{2 \times 60 \times 20}{60 + 20} = \frac{2400}{80} = 30$$

带入到机器学习中：

精确率和召回率，在数学本质上类似于“速度”这种比率（Rate）。如果你的模型有一个指标极其糟糕（比如 $R=0.02$），说明模型在绝大部分真实样本上都在“龟速行驶”。调和平均数会毫不留情地把总分拽向那个极低的数值，从而揭穿模型的真实水平。

四、 具体的极端数值示例

我们来看看算术平均是如何“骗人”的，而 F1 又是如何“打假”的。

示例 1：极端偏科的“躺平”模型
模型极其保守，只抓了 1 个确定性最高的正样本，其他全判定为负。

  • 精确率 $P = 1.0$ (百发百中)
  • 召回率 $R = 0.02$ (漏掉了 98% 的目标)
  • 算术平均：$\frac{1.0 + 0.02}{2} = 0.51$ （看起来模型勉强及格了！）
  • F1 分数 (调和平均)：$\frac{2 \times 1.0 \times 0.02}{1.0 + 0.02} = \frac{0.04}{1.02} \approx 0.039$ （原形毕露，不及格！）

示例 2：均衡发展的优秀模型

  • 精确率 $P = 0.8$
  • 召回率 $R = 0.8$
  • 算术平均：$\frac{0.8 + 0.8}{2} = 0.8$
  • F1 分数 (调和平均)：$\frac{2 \times 0.8 \times 0.8}{0.8 + 0.8} = \frac{1.28}{1.6} = 0.8$

结论： 当 $P$ 和 $R$ 一样大时，F1 和算术平均是一样的。但只要它们拉开差距，F1 就会狠狠地惩罚那个较小的值。

动手验证：F1 是如何惩罚“偏科”模型的
算术平均 (虚假的及格分) 0.550
公式：(P + R) / 2
F1 分数 (真实的判别力) 0.182
公式：2 * P * R / (P + R)
严重偏科！ 算术平均给出了 0.55 的幻觉，但 F1 分数被短板（R=0.10）死死拖拽至 0.18。这就像以 100km/h 的速度开了一小段，剩下的路程全在以 10km/h 爬行，整体平均速度注定极其缓慢。

💡 什么是多分类的“聚合”？（考试比喻）

在二分类（比如只判断是不是猫）中，算一个精准率就够了。但现在模型做的是多分类（比如要同时认出猫、狗、兔子）。这就好比一个学生参加了 3门不同科目 的期末考试。

有的科目题目很多（比如语文 90题），有的极少（比如物理 2题）。

模型在每个科目上都有自己的单科成绩（局部的 $P_k, R_k, F_k$）。但老板/老师只要一个“最终总评得分”。我们要怎么把这3门课的成绩合并成1个数字？这就是下面三种聚合算法要解决的现实问题。
